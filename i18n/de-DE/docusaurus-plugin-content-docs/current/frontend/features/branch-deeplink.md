# Branch Deeplink

## Was es tut

Branch Deeplink erweitert die Verknüpfungsfunktionen der Atlas-Plattform mithilfe der **Branch.io SDK** – einer Drittanbieter-Tieflink- und Attribution-Plattform, die weit über mobile Apps genutzt wird.

Wo KFC Universal Links Nutzer zu In-App-Inhalte vernetzen, fügt Branch oben eine Ebene der Intelligenz hinzu: es verfolgt *woher* ein Benutzer kam, behandelt tiefe Links für Benutzer, die noch nicht die App haben, und liefert Zuschreibungsdaten, die Marketingaktivitäten mit App-Installationen und Aufträgen verbinden.

## Schlüsselfähigkeiten

| Kapazität | Warenbezeichnung |
|---|---|
| **Deferred Deep Linking* | Ein Benutzer klickt auf einen Branch-Link, installiert die App und wird beim ersten Start auf den richtigen In-App-Bildschirm gebracht – obwohl die App nicht installiert wurde, wenn sie geklickt haben |
| ** Beitrag** | Links verfolgen, welche Kampagne, Kanal oder kreativ die Installation oder Sitzung fuhren |
| **Cross-Plattform-Routing* | Bremse behandelt iOS, Android, und Web-Rückschläge aus einem einzigen Link |
| **Unified Link Management** | Marketing-Teams können tiefe Links über das Branch Armaturenbrett ohne Engineering-Einbindung erstellen und verwalten |

## Wie es auf Universal Links reagiert

KFC Universal Links und Branch Deeplinks dienen komplementären Zwecken:

|  | Universal Link | Branch Deeplink |
|---|---|---|
| **Primäreinsatz** | Direktes Routing (app installiert) | Beitrag + latente Verknüpfung |
| **App nicht installiert** | Zurück zur Übersicht | Kann auslösen App Store redirect + deferred tief link |
| **Markenzuschreibung* | Nein | Ja. |
| **Verwaltet von** | Ingenieurwesen | Marketing und Marketing |
