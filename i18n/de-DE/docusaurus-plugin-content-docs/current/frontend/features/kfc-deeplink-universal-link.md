# KFC Deeplink | Universal Link

## Was es tut

Mit der KFC Universal Link Funktion können unterstützte KFC-URL direkt innerhalb der KFC-Mobilfunk-App geöffnet werden – der Benutzer direkt auf den entsprechenden In-App-Bildschirm anstatt auf einer Browser-Seite.

Wenn die **app installiert ist*, unterstützte Routenbenutzer direkt auf den passenden Bildschirm (z.B. eine Produktseite, eine Promo oder eine Restaurantlandung). Wenn die **app nicht installiert ist*, fällt derselbe Link auf das Weberlebnis anmutig zurück.

Das Ergebnis ist ein nahtloser, reibungsloser Weg von jedem Touchpoint – eine Marketing-E-Mail, ein QR-Code in-Restaurant, eine soziale Anzeige, eine Push-Benachrichtigung – direkt an den richtigen Ort in der App.

## Warum es aufpasst

Ohne Universal Links öffnet jeder externe Link zu KFC-Inhalte in einem Browser, auch wenn der Kunde die App installiert hat. Das schafft unnötige Reibung – und verlorene Umwandlungsmöglichkeiten.

Universal Links schließen diese Lücke. Die URL bleibt gleich. Das Ziel passt sich an das, was der Kunde installiert hat.

## Anwendungsbereich

| Gebiet | Detail |
|---|---|
| **Unterstützte Domains** | KFC-eigene URLs pro Markt konfiguriert |
| **Routing** | Path-basiert — jede URL-Pfadkarte zu einem In-App-Ziel |
| **App installiert** | Der Benutzer wird direkt zum In-App-Bildschirm gebracht |
| **App nicht installiert** | Der Benutzer wird an das entsprechende Weberlebnis geschickt |
| ** Formulare** | iOS, Android |

## Eintrittspunkte

Universal Links werden typischerweise ausgelöst von:

- Marketing-E-Mails und SMS Kampagnen
- QR Codes (im Restaurant, Verpackung, im Freien)
- Social Media Anzeigen und organische Beiträge
- Push-Benachrichtigungen
- Web-to-App-Handoff-Ströme
