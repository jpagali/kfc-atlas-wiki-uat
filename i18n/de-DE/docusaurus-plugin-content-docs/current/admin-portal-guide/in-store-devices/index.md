# In-Store Geräte

Dieser Abschnitt umfasst die Verwaltung von POS-Terminals und Kioske in KFC-Restaurants, einschließlich Geräteaufbau, Status und Authentifizierung.

:::note Byte POS Caveat
Die unten stehenden Geräteführer werden für Märkte mit **Byte POS** geschrieben.

Wenn ein Markt nicht Byte POS verwendet, muss **Byte Connect** im Byte Commerce-An Bordbündel enthalten sein, und Teams sollten den Bestellpfad als **Byte Commerce -> Byte Connect -> POS** anstatt direkte Kommunikation von Byte Commerce auf den Markt POS zu übernehmen.
:::

## Themen

- [Ein-Zeit-Passwort generieren](/docs/admin-portal-guide/in-store-devices/generate-one-time-password/)— Erstellen Sie einen temporären Authentifizierungscode für Geräteaufbau oder Wieder-Authentifizierung.
- [In-Store-Gerät Details anzeigen](/docs/admin-portal-guide/in-store-devices/view-in-store-device-details/)— Vollständige Details eines Geräts anzeigen, einschließlich Name, Typ, Status und Menü, Geschichte veröffentlichen.
- [In-Store deaktivieren](/docs/admin-portal-guide/in-store-devices/deactivate-in-store/)— Deaktivieren Sie ein Gerät vom Empfang von Menü-Updates oder Bearbeitungsaufträgen.

Brauchen Sie den Hintergrundkontext? Weiterlesen[Byte Connect](/docs/byte-capabilities/enablement/byte-connect).
