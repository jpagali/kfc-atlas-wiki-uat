# 📋 Plattform Glossar

> Definitionen für alle Begriffe, die in der Dokumentation von Byte Capabilities verwendet werden. Wenn im Zweifel, was ein Begriff bedeutet, beginnen Sie hier.

---

| Begriff | Begriff |
|---|---|
| **Byte Helium** | Yum! Unternehmen Design System und Front-End-Stiftung. KFC Atlas wird auf Helium gebaut und gehäutet, um die Marke KFC zu reflektieren |
| **Byte Commerce** | Der Handelsmotor hinter Atlas. Es verarbeitet Aufträge, Preise, Zahlungen und POS-Injektion; wenn ein Markt nicht auf Byte POS ist Byte Connect als Brücke zum Markt POS erforderlich |
| * * * * | Die Backend-Komponente von Byte Commerce — siehe Byte Commerce |
| **Byte Portal* | Die Admin-Schnittstelle kombiniert mit Byte Commerce für Markt- und Ops-Teams, um die Plattform zu konfigurieren und zu betreiben |
| **Byte Menu** | Das Menü Autoring-Tool — extern nach Atlas; Byte Portal vergibt veröffentlichte Menüs |
| **Byte Connect** | Die mittlere Schicht, die verwendet wird, wenn Atlas / Byte Commerce eine nicht-Byte POS-Umgebung erreichen muss; es unterstützt auch Aggregatorauftrag fließt in Byte Commerce |
| **PSP** | Payment Service Provider — übernimmt Kartenmarkierung und Erfassung |
| **Vault** | Sichere Aufbewahrung für Kartentoken und PSP-Anmeldeinformationen |
| **PLP** | Produkt Listing Page — das Menü durchsuchen Ansicht |
| **PDP** | Produkt Detailseite — Einzelteilansicht mit Optionen |
| **OTP** | One-Time PIN — für passwortloses Sign-in verwendet |
| **SSO** | Single Sign-On — zentraler Authentifizierungsservice |
| **CMP*** | Consent Management Platform — verwaltet Cookies und Marketing-Einwilligungen |
| **CRM** | Customer Relationship Management — verwaltet Kundenprofile und komms prefs |
| **KDS** | Küchendisplay System — der Bildschirm in-store zeigt eingehende Bestellungen |
| **POS** | Point of Sale — das in-store order management system; Atlas erreicht Byte POS durch Byte Commerce und erreicht nicht-Byte POS Märkte durch Byte Commerce plus Byte Connect |
| **DCI** | Data Contracts & Interfaces – die versionierte Registrierung aller API/Event-Schema |
| **RBAC** | Role-Based Access Control — wie Portal Benutzerrechte verwaltet werden |
| **Scope Guard** | Byte Portal Sicherheitsmechanismus, der den Zugriff auf UI und API-Ebene blockiert |
| **Add-Hope** | Das weltweite Spendenprogramm von KFC im Checkout integriert |
| **Split Tender** | Bezahlung mit zwei Zahlungsarten — z.B. Gutschein + Karte |
| **Idempotency Key** | Eine einzigartige ID, die doppelte Bestellungen oder Zahlungen verhindert, wenn eine Anfrage abgerufen wird |
| **TR** | Technische Anforderungen Dokument |
| **FR*** | Funktionsanforderungen Dokument |
| **SLO*** | Ziel der Service Level — das Leistungsziel eines Systems |
| **DLQ** | Dead Letter Queue — wo fehlgeschlagene Webhook-Veranstaltungen stattfinden für die Bewertung |
| **3DS** | 3D Secure — Kartenauthentifizierungsprotokoll zur Betrugsprävention |
| **Byte** | Der Programmname für die gesamte E-Commerce-Initiative KFC |
| **Atlas** | KFCs globales Frontend für digitale Bestellung, gebaut auf Byte Helium und betrieben von Byte Commerce + Byte Portal, mit Byte Connect enthalten, wo immer ein Markt POS ist nicht Byte POS |
| **Kontext** | Der Rekord der Plattform von Markt, Sprache, Bestellmodus und Laden eines Kunden |
| **BCOM** | Kurzform von Commerce Backend — siehe Commerce Backend |
| **Patch** | Ein Menü überschreiben, das oben auf einer veröffentlichten Menü-Version angewendet wird — für Preis- oder Verfügbarkeitsänderungen auf Speicher- oder Gruppenebene verwendet |
| ** Store Group** | Eine logische Gruppierung von Geschäften in Byte Portal verwendet für Menüzuweisung, Promotions scoping und Steuerregel Erbschaft |
| **Coverage Zone** | Eine im Byte Portal / Geo-Service konfigurierte Liefergeografische Grenze, die bestimmt, welche Adressen Lieferaufträge empfangen können |
| **Feature Flag** | Ein Laufzeit-Toggle in Byte Portal, die ein Feature für einen bestimmten Markt, Kanal oder Kundensegment ohne Umsetzen ermöglicht oder deaktiviert |
| **Aggregat** | Eine Drittanbieter-Lieferplattform (z.B. Uber Eats, DoorDash), deren Bestellungen über Byte Connect in Byte Commerce aufgenommen werden |
