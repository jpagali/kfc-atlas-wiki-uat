# ✅ 市場ローンチチェックリスト

> このチェックリストは [現実チェック](/docs/byte-capabilities/enablement/reality-check) に基づいています。ローンチで実際に起こる依存関係不足や失敗パターンを前提に、順番に確認してください。Phase 1 が完了するまでは UAT を開始しないでください。

---

## このチェックリストの使い方

各項目には **リスクレベル** と **Go / No-Go 判定** があります。🔴 は絶対的なブロッカーで、完了しない限り安全にローンチできません。🟡 は UAT 前に確認すべき項目です。🟢 はローンチ前に仕上げるべき項目です。

---

## Phase 1 - 基盤（テスト開始前に必須）

### 1.1 Byte Portal アクセスとユーザー

:::danger Go / No-Go 判定
チームへ適切な Byte Portal 権限が付与されるまでテストは開始できません。
:::

- [ ] 市場向け Byte Portal 環境が用意されている
- [ ] 市場管理ユーザーが招待され、正しい RBAC 権限が付与されている
- [ ] Scope Guard を確認し、市場外データへアクセスできない
- [ ] すべての Byte Portal ユーザーで MFA が有効

---

### 1.2 店舗設定

:::danger Go / No-Go 判定
店舗設定が欠けている、または誤っていると顧客は注文できません。
:::

- [ ] すべての対象店舗が Byte Portal に正しい店舗情報で作成されている
- [ ] 営業時間が **正しい現地タイムゾーン** で入力されている
- [ ] 店舗のリアルタイム状態が POS と連携し、open / paused / closed が反映される
- [ ] Store group が正しい階層で設定され、循環がない
- [ ] 必要に応じてスロットリングポリシーが設定されている

---

### 1.3 メニュー

:::danger Go / No-Go 判定
公開パイプラインが完了していないと顧客にメニューが表示されません。
:::

- [ ] メニュー内容が **Byte Menu** で作成済み（Portal ではない）
- [ ] ローンチ対象チャネル向けに Byte Menu でメニューバージョンが公開済み
- [ ] Byte Portal でそのメニューバージョンが市場へ割り当て済み
- [ ] 店舗 / チャネルごとに、商品、価格、モディファイアが正しく表示されることを確認済み
- [ ] 配達専用 / 受取専用商品がチャネルごとに正しく設定されている
- [ ] 必要市場では栄養情報とアレルゲン情報が Byte Menu に入力済み

---

### 1.4 税設定

:::danger Go / No-Go 判定
税が誤っていると、チェックアウト価格の誤りと法務リスクにつながります。
:::

- [ ] 市場向け税プロファイルが正しい税率とカテゴリで作成されている
- [ ] Finance 承認のもと、現地税法との整合が確認されている
- [ ] Byte Portal で税プロファイルが市場へ割り当て済み
- [ ] エンドツーエンドのチェックアウトで税内訳が正しい

---

### 1.5 決済 / PSP

:::danger Go / No-Go 判定
PSP の設定が end-to-end で完了していないと決済は処理できません。
:::

- [ ] 市場向け PSP 契約が整っている
- [ ] PSP 認証情報が Vault に保存されている（Byte Portal に直接置かない）
- [ ] Byte Portal に PSP プロファイルがあり、必要な支払い手段が有効
- [ ] チャネル / 市場単位のルーティングルールが設定済み
- [ ] 必要に応じて追加料金と手数料ポリシーが設定済み
- [ ] ステージングでカード決済を end-to-end で確認済み

---

## Phase 2 - チャネルと体験（UAT 前に必須）

### 2.1 ローカライズ

:::warning UAT 前ゲート
ローカライズが誤っていると、顧客へ誤メニュー、誤価格、誤文言が表示されます。
:::

- [ ] Byte Portal / Config service で市場とロケールが正しく設定されている
- [ ] 言語 / ロケール判定（geo-IP と明示選択）が確認済み
- [ ] すべての UI 文言に翻訳キーが揃っている
- [ ] 欠落翻訳時に空白ではなく英語へ正しくフォールバックすることを確認済み

---

### 2.2 注文モード

:::warning UAT 前ゲート
注文モードは入口です。ここが失敗すると注文自体ができません。
:::

- [ ] 受取モードを end-to-end で確認済み: 店舗選択 → メニュー → チェックアウト → 注文完了
- [ ] デリバリーモードを end-to-end で確認済み: 住所入力 → 配達範囲確認 → メニュー → チェックアウト → 注文完了
- [ ] デリバリー有効市場では、配達範囲が Byte Portal / Geo service に設定済み
- [ ] デリバリー非対応市場では、デリバリーモードが無効になっている
- [ ] 閉店店舗では注文不可になることを確認済み

---

### 2.3 Identity とアカウント

- [ ] OTP メールサインイン（登録とログイン）が確認済み
- [ ] Google OAuth を確認済み（市場で有効な場合）
- [ ] Apple Sign-In を確認済み（市場で有効な場合）
- [ ] プロフィール編集、連絡設定、アカウント削除が動作する
- [ ] ゲストチェックアウト導線が確認済み

---

### 2.4 プロモーション

:::warning UAT 前ゲート
対象範囲が誤ったプロモーションは、想定外の市場やチャネルで使われてしまいます。
:::

- [ ] すべてのローンチ用プロモーションが Byte Portal に正しく設定されている:
  - 市場スコープ
  - チャネルスコープ
  - 開始日 / 終了日
  - 予算上限
- [ ] プロモコードを確認済み。無効コード時はエラーを返してもカートは保持される
- [ ] 意図しない限り他のプロモーションと重複適用されない

---

### 2.5 ロイヤルティ（ローンチ時に有効な場合）

- [ ] Global Loyalty チームと連携し、市場でロイヤルティが有効化済み
- [ ] 新規登録向けロイヤルティ導線を確認済み
- [ ] 注文後にポイント残高が正しく反映される
- [ ] 市場に正しい特典カタログが表示される
- [ ] 特典利用を end-to-end で確認済み
- [ ] 必要に応じてチャレンジが設定済み
- [ ] 誕生日 / 登録特典トリガーが設定済み

---

### 2.6 配達追跡（リアルタイム追跡がある場合）

- [ ] 3P logistics 連携が確認済みかつテスト済み
- [ ] 注文追跡画面でリアルタイム ETA が正しく表示される
- [ ] 追跡イベント遅延時のフォールバック動作が確認済み

---

## Phase 3 - コンテンツと法務（ローンチ日までに必須）

### 3.1 CMS コンテンツ

:::warning これらは初日から顧客へ見えます。
:::

- [ ] ホームページコンテンツが CMS で作成・公開済み
- [ ] About / Our History ページが公開済み
- [ ] FAQ コンテンツが作成・公開済み
- [ ] 必要に応じて Careers / job board が設定済み

---

### 3.2 法務とコンプライアンス

- [ ] Terms & Conditions が対象ロケールと施行日で Legal CMS に公開済み
- [ ] Privacy Policy が対象ロケールで公開済み
- [ ] 現地 Cookie 法に従って CMP にカテゴリ設定済み
- [ ] Cookie consent banner で設定が正しく保存されることを確認済み

---

## Phase 4 - 運用（本番承認前）

### 4.1 Feature Flags

- [ ] 市場固有の feature flag がすべて確認され、Byte Portal で正しく設定されている
- [ ] 重大機能向けの kill-switch が識別されている
- [ ] ステージングで想定外の有効 / 無効化がないことを確認済み

---

### 4.2 Webhook と外部連携（必要な場合）

- [ ] 外部システム向け webhook 購読が設定済み
- [ ] 購読先 endpoint が冪等に重複イベントを処理できることを確認済み
- [ ] DLQ が監視され、アラートが設定済み

---

### 4.3 返金と運用

- [ ] Byte Portal に返金ポリシー（上限、期限、承認ルール）が設定済み
- [ ] 運用チームが返金処理に必要な Byte Portal 権限を持っている
- [ ] 役割ベースの PII アクセスで注文検索を確認済み

---

### 4.4 可観測性

- [ ] 市場向けヘルスチェックと SLO ダッシュボードが設定済み
- [ ] チェックアウト、決済、追跡向けアラートルールが有効
- [ ] コンプライアンスチームが監査ログへアクセスできる
