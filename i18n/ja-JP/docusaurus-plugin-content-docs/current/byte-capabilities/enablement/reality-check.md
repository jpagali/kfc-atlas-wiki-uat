# ⚠️ 現実チェック - 市場チームが知っておくべきこと

> 最も高くつくローンチ問題は、機能不足ではなく誤解から起こります。本番前に必ず確認してください。

---

## よくある大きな誤解

ローンチ時にチームが持ち込みがちな前提と、その実際のところを整理します。

### ❌ "Byte Helium が価格を持っている"

**現実:** Byte Helium は価格を表示するだけで、保持も計算も制御もしません。価格は Commerce Backend と Menu Service 側の責務です。アプリで価格が誤っている場合、修正先はバックエンドまたはメニュー設定です。

### ❌ "チェックアウトフローは Byte Helium で設定できる"

**現実:** チェックアウトフロー（バリアント、手順、手数料）は Commerce Backend の Checkout Orchestrator が制御します。市場は Byte Portal 設定でどのバリアントを使うかを定義し、Byte Helium はその結果を描画します。

### ❌ "Byte Portal でメニューを作成できる"

**現実:** Byte Portal は公開済みメニューの割当とパッチ適用を行うだけです。メニュー作成（商品、説明、価格の定義）は **Byte Menu** で行います。Byte Portal はメニューをゼロから作れません。

### ❌ "プロモーションは Byte Helium で設定する"

**現実:** プロモーションルール、適用可否、コード、予算はすべて **Byte Portal** で設定します。Byte Helium は Promo Engine が返す内容を表示するだけで、適用可否は常にサーバー側で判定されます。

### ❌ "リアルタイム配達追跡は常に使える"

**現実:** 配達追跡は **市場ごとに任意** です。サードパーティ物流連携が必要で、これがない場合は POS の注文状態しか見えません。

### ❌ "市場がローンチすればロイヤルティは自動で動く"

**現実:** ロイヤルティには 3 つ必要です。1. 市場向けプログラム設定、2. 顧客のサインイン、3. 顧客の登録。ゲストユーザーにはロイヤルティ特典はありません。

## 見えにくい依存関係

次の機能は、事前に別のものが整っていないと、実行時に失敗するか、そもそも表示されません。

| 機能 | 裏で依存しているもの | 足りない場合の影響 |
|---|---|---|
| プロモ適用可否 | 市場ごとのサーバー側 Promo Engine | オファーは見えても利用時に失敗 |
| 配達追跡 | 市場ごとの 3P logistics 連携 | 追跡が unavailable 表示になる |
| 保存済み住所 | 顧客のサインイン | ゲストは毎回住所が消える |
| 注文履歴 | 顧客アカウントが存在し連携済み | 履歴に注文が出ない |
| チャレンジとゲーミフィケーション | 市場でロイヤルティが有効 | Challenges タブが出ない |
| Cookie 同意 | 市場向け CMP 設定 | 現地法に適合しない可能性 |
| Webhook 購読先 | 購読側の冪等処理 | イベントが二重処理される |
| 返金処理 | Byte Portal の返金ポリシー設定 | 返金リクエストが検証失敗 |
| チップ機能 | デリバリー有効 + PSP の別キャプチャ設定 | すべての市場では使えない |
| 税の正確性 | Byte Portal の税プロファイル割当 | チェックアウトで誤税額や失敗 |

## ローンチ時の典型的な失敗ポイント

### 店舗設定の問題
営業時間が誤タイムゾーンで設定されている、またはリアルタイム店舗状態が POS と連携していない。その結果、閉店店舗へ顧客が注文しようとする。

### メニュー未公開
Byte Menu では作成済みでも、Byte Portal で市場へ割り当て / 公開されていない。その結果、顧客にメニューが出ない。

### プロモコードのスコープ誤り
市場 / チャネル / 日付スコープが不正。その結果、本来ローカル向けのプロモがグローバルに見える。

### PSP 認証情報が Vault にない
Byte Portal 設定はあるが PSP 秘密情報が Vault に未設定。その結果、カード決済が静かに失敗する。

### ロケール / 翻訳が未完了
翻訳キーが欠落したまま市場ローンチし、英語フォールバックや空欄が出る。

### 配達範囲未設定
デリバリーを有効にしたが、カバーゾーンが未設定。その結果、すべての住所が拒否される。

### 市場でロイヤルティ未有効
Byte Helium に UI は出るが、Loyalty service で有効になっていない。その結果、登録が失敗する。

## 市場が苦労しやすい領域

| 領域 | リスクレベル | 取るべき行動 |
|---|---|---|
| 税設定 | 🔴 高 | Go-live 前に Finance と Portal の税担当と確認する |
| メニュー公開フロー | 🔴 高 | Byte Menu と Portal チームで publish → assign → preview → channel publish の流れを揃える |
| PSP / 決済設定 | 🔴 高 | PSP 契約、Vault 秘密情報、ルーティング、追加料金を checkout テスト前に準備する |
| 配達追跡 | 🟡 中 | リアルタイム追跡を使うかを早めに決める |
| ロイヤルティ有効化 | 🟡 中 | Global Loyalty チームとの調整が必要 |
| CMS コンテンツ準備 | 🟡 中 | About、FAQ、法務ページはローンチ前に作成が必要 |
| Feature flag 設定 | 🟡 中 | 市場で何が有効かを確認し、テストする |
| 翻訳完全性 | 🟢 低（ただし目立つ） | UAT 前に翻訳カバレッジを確認し、欠落は英語フォールバックを確認する |
