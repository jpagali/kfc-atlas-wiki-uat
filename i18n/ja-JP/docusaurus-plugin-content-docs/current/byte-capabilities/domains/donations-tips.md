# 寄付とチップ

**何をするものか:** チェックアウト中に、顧客が慈善寄付（Add Hope）や配達ドライバーへのチップを追加できるようにします。

**なぜ重要か:** Add Hope は KFC にとって世界的に重要な CSR プログラムです。チップはドライバー満足度や配達品質の印象向上につながります。どちらも慎重な決済処理と市場ごとの有効化が必要です。

---

## 機能テーブル

| 機能 | 解決する課題 | 何をするか | 仕組み | 依存先 | 前提条件 | 制約 | 影響を受ける指標 |
|---|---|---|---|---|---|---|---|
| **Add Hope 寄付** | 顧客がチェックアウト時に寄付したい | 注文合計へ寄付額を追加できるようにする | Donation service が寄付先候補を返し、顧客が金額や端数切り上げを選ぶと、寄付行が注文へ追加される | Donation service、PSP | チェックアウトが有効で、市場で Add Hope が有効化されている | 市場側で Add Hope を有効にする必要あり。Donation service が失敗した場合（5xx）は寄付をスキップし、注文にフラグを付ける | 寄付付与率、平均寄付額 |
| **配達ドライバーへのチップ** | 顧客がドライバーへチップを渡したい | 配達注文にチップ額を追加できるようにする | Tip service が注文へ tip tender を付与し、市場によっては PSP で別決済となる | Tip service、PSP | デリバリーモードが有効で、チェックアウト進行中 | チップはデリバリー注文のみ利用可。別決済キャプチャになる場合がある | チップ付与率、平均チップ額 |
| **Add Hope について見る** | 顧客が寄付先の内容を知りたい | Add Hope の説明ページを表示する | CMS ページを取得して表示する | CMS service、Donation config | なし | コンテンツ管理は CMS 側で、Byte Helium ではない | ページ閲覧率、寄付転換率 |

---

## 技術ソース

<details>
<summary>📎 技術ソース: Add Hope 寄付 / ドライバーチップ</summary>

- **FRD 参照:** FRD-HEL-022, FRD-HEL-023, FRD-HEL-073
- **TRD ドメイン:** Donations & Tips
- **主要インターフェース / API:** Donation Config / Collect、Add Tip（Bearer auth）
- **データ契約:** DonationLine（amount、causeId）、TipTender（amount、method）
- **要点:**
  - Add Hope は Byte Portal の市場設定で有効化される
  - Donation service が失敗した場合（5xx）は寄付をスキップし、注文へレポート用タグを付与
  - チップは市場や PSP によって別キャプチャになる場合がある
  - コンプライアンス対応のため、寄付額を記録する報告項目が必要
  - API は加算的変更のみ、90 日の非推奨期間あり

</details>
