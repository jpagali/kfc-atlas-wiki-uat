# 🛠 Byte Portal の機能

**Byte Portal** は、Byte Commerce と対になる管理コントロールプレーンです。市場チームと運用チームはこのツールを使って、Atlas フロントエンドの背後にあるプラットフォーム設定と運用を管理します。Byte Portal がなければ市場は本番ローンチできません。顧客が Atlas で体験するあらゆる機能は、その前段で Byte Portal に正しく設定されていることが前提です。

Byte Portal 自体はメニューを作成しません（それは Byte Menu の役割です）。また、決済を処理するわけでもなく（PSP の役割）、プッシュ通知を送信するわけでもありません（通知プロバイダーの役割）。Byte Portal は、それ以外の設定、割り当て、制御を担います。

---

## Byte Portal が担う領域

| 機能 | 市場チームがここで行うこと |
|---|---|
| **ユーザーと権限（RBAC）** | ユーザー招待、権限付与、市場 / 店舗 / グループ範囲の制御 |
| **店舗とグループ設定** | 店舗作成、営業時間、リアルタイム状態、POS マッピング、スロットリング設定 |
| **メニュー割り当てとパッチ** | 公開済みメニューの割り当て、価格 / 提供可否 / デイパート上書きの適用 |
| **プロモーション設定** | 適用条件、コード、予算、対象範囲付きのプロモーション作成 |
| **税設定** | 市場ごとの税プロファイル、税率、ルール、端数処理の定義 |
| **決済設定** | PSP プロファイル、支払い手段、ルーティングルール、追加料金ポリシーの設定 |
| **注文と返金（運用）** | 注文検索、返金処理、調整管理 |
| **ローカライズとコンテンツ管理** | ロケール、翻訳キー、コンテンツブロック、法務表示の管理 |
| **機能フラグと設定** | 市場、チャネル、顧客セグメント単位での機能展開制御 |
| **レポートとエクスポート** | レポート定義、ビュー保存、オンデマンド / 定期エクスポート |
| **Webhook と外部連携** | 外部システムへのイベント購読設定 |
| **監査と可観測性** | 管理操作の監査ログ、ヘルスチェック、SLO 監視 |

:::tip Byte Portal の実務ガイド
Portal 操作の手順ガイドは [Admin Portal Guide](/docs/admin-portal-guide/) を参照してください。
:::

:::caution Byte Portal はメニューを作成しない
メニュー内容（商品、説明、価格、オプション）は **Byte Menu** という Atlas 外の別ツールで作成されます。Byte Portal は公開済みメニューバージョンを割り当て、必要に応じてその上にパッチを適用できますが、メニューをゼロから作ることはできません。
:::
