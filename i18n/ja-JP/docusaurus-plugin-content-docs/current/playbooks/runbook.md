# Braze 歓迎特典キャンバス

このプレイブックでは、KFC チームが Braze で新規顧客向けの歓迎特典キャンバスを設定する方法を説明します。現時点ではコンテンツ自体は検証済みとみなし、構成、配信タイミング、実装フローに集中します。

---

## 目的

KFC が新規または新規識別済みの顧客に対して、自動の歓迎特典ジャーニーを配信したい時に使います。

よくある例:

- アプリ新規登録特典
- ロイヤルティ登録直後の特典
- アカウント作成後の初回購入促進特典

## 事前準備

- 検証済みの reward 内容と offer ロジック
- KFC ブランド承認済みの push / email / in-app 文言
- 合意済みの audience 定義
- Braze 上で canvas を作成または編集できる権限
- 遷移先リンクや deeplink の確認

## ステップ1: エントリー audience を定義する

KFC の歓迎施策でよくある audience 例:

- アプリ登録完了ユーザー
- ロイヤルティ参加完了ユーザー
- 登録時にマーケティング同意した新規顧客

既存ユーザーが誤って再度このジャーニーへ入らないように条件を確認します。

## ステップ2: トリガーと遅延を決める

決める内容:

- キャンバス開始の trigger event
- 初回メッセージ送信までの delay
- 即時配信にするか、少し待ってから配信するか

KFC の歓迎特典では、トランザクション通知に見えすぎないよう、短い遅延が有効な場合があります。

## ステップ3: メッセージ構成を決める

推奨構成:

1. welcome message
2. reward explanation
3. KFC アプリまたは注文導線への CTA
4. 未利用時の reminder（必要なら）

最初のメッセージはシンプルでブランド感のあるものにし、特典内容は一目で理解できるようにします。

## ステップ4: 特典ロジックを紐づける

確認項目:

- 顧客が受け取る reward の内容
- 有効期限
- 特定メニュー、カテゴリ、またはランディングページへの deeplink の有無
- 市場別除外条件の有無

もし reward が Atlas 側の live promotion に依存する場合は、Braze 公開前に対象 promotion が正しい店舗へ割り当てられていることを確認してください。

## ステップ5: 顧客ジャーニーを QA する

公開前に次を確認します。

1. audience に正しく入るか
2. メッセージ配信タイミングが意図通りか
3. 文言、ブランド表現、リンク先が正しいか
4. reward が利用できない場合の fallback があるか
5. CTA が期待した KFC 体験へ遷移するか

## ステップ6: 公開後のモニタリング

公開後は次を確認します。

- エントリー数
- 配信率
- app / ordering 導線へのクリック率
- reward 利用率
- サポート問い合わせや顧客の混乱

## KFC 向けメモ

- 文言とトーンは KFC の最新ブランドキャンペーンに合わせる
- 歓迎特典は理解しやすく、利用しやすい設計にする
- 市場別施策の場合は、Braze audience 条件と Atlas promotion 対象範囲が完全に一致していることを確認する
