# リリース 2026.03.19

**2026年3月19日** にリリースされました。

このリリースでは、Atlas Wiki の用語とナビゲーションを整理し、Byte ドキュメント群に daypart とプラットフォーム参照コンテンツを追加し、テーマ追従型のプラットフォーム階層埋め込みを導入しました。また、3月19日初回ビルドで加わった Atlas Peek プロトタイプとデスクトップナビバーの修正も引き継いでいます。

## ハイライト

- **Atlas Peek プロトタイプがライトモードで正しく表示されるように改善**
  PICKUP / ASAP チップ、トップバー、ナビゲーション、ジャーニーシェルバーが、ダークモードの見た目に引っ張られず、正しいライトモード配色で表示されるようになりました。

- **重複していたサイドバーカテゴリ名を解消**
  Admin Portal Guide のサイドバーに、Stores、Menus、Products など同名のネストカテゴリが存在していました。これらを `Tasks` サブカテゴリへ改名し、翻訳衝突を避けつつ見分けやすくしました。

- **ナビバーがブラウザのズームレベルに追従**
  ビューポートが狭くなるにつれてナビバー検索欄が段階的に縮小し、一定幅以下では非表示になるため、ズーム時でも主要ナビリンクを見失いにくくなりました。

- **デスクトップナビバーから Contribute ボタンを削除**
  ナビバーの混雑を減らすため、デスクトップの Contribute ボタンを外しました。寄稿スタジオにはフッター CTA から引き続きアクセスできます。

- **検索プレースホルダーを更新**
  ナビバー検索欄の文言を、より会話的な `"What's on your mind?"` に変更しました。

- **ヒーロー検索バーの幅を調整**
  ヒーロー検索バーの最大幅を 720px から 600px に縮め、全体の視覚バランスを改善しました。

- **Byte Capabilities のクレジットを更新**
  クレジットページを `The Byte Capabilities` 表記に寄せ、元の Confluence ドキュメントへのリンクを追加しました。

- **Byte 関連の用語と導線を横断的に整理**
  `Byte Helium`、`Byte Commerce Backend`、`Byte Portal` を基準に名称と導線をそろえ、プラットフォーム構造を一目で理解しやすくしました。

- **Byte Portal Guide で daypart を通しでカバー**
  Menus セクションに、daypart 概要、作成、店舗別カスタマイズ、価格 / 提供可否、カテゴリ表示設定のガイドを追加しました。

- **Byte Capabilities により明確な入口を追加**
  Byte Helium、Byte Commerce Backend、Byte Portal 用の概要ページを追加し、非 Byte POS 市場向けに Byte Connect の説明ページも新設しました。

- **プラットフォームメンタルモデルをテーマ同期の埋め込みへ移行**
  以前の静的な階層図を、サイトテーマに追従する埋め込み図へ差し替え、Byte Connect から市場 POS までの流れをより明確にしました。

## 変更内容

### Atlas Peek プロトタイプ

- `.journey-shell-chip`（PICKUP / ASAP チップ）に、黒文字と透明背景のライトモードスタイルを追加
- `.journey-shell-bar` にライトモード背景（白背景）を追加
- `.topbar` と `.web-nav` にライトモードスタイル（白背景）を追加
- アクションシート内 `.pill` ボタンへライトモードスタイルを追加

### サイドバー ナビゲーション

- 重複していた `Stores` のネストカテゴリを `Store Tasks` に改名
- 重複していた `Menus` のネストカテゴリを `Menu Tasks` に改名
- 重複していた `Products` のネストカテゴリを `Product Tasks` に改名
- 重複していた `Promotions` のネストカテゴリを `Promotion Tasks` に改名
- 重複していた `Bundles` のネストカテゴリを `Bundle Tasks` に改名
- `Promotions` 配下の重複 `Store Groups` カテゴリを `Store Group Tasks` に改名

### ナビバー

- 画面幅に応じてナビバー検索欄を縮小するレスポンシブ条件を追加:
  - 1500px 以上: テーマ切り替えを非表示
  - 1400px 以上: Contribute ボタンを非表示
  - 1300px: 検索欄を 220px に縮小
  - 1200px: 検索欄を 180px に縮小
  - 1100px: 検索欄を 140px に縮小
  - 1000px: 検索欄を完全に非表示
- `docusaurus.config.js` のナビバー項目から contribute を削除
- ナビバー右側アイテムの間隔を 0.75rem から 0.5rem に縮小
- ナビバー検索欄の最大幅を 220px から 280px に拡大

### 検索

- ナビバー検索プレースホルダーを `"Search articles and headings"` から `"What's on your mind?"` に変更
- 日本語プレースホルダーを `"気になることは？"` に更新
- ヒーロー検索バーの最大幅を 720px から 600px に変更

### ドキュメント

- Byte Capabilities クレジットページのタイトルと説明を更新
- Varima Henry と Wesley Weidemann のクレジットに Confluence リンクを追加

### ドキュメント構成

- メインドキュメントナビバーの並びを Byte Capabilities、Byte Portal Guide、Front-End Guide、Playbooks の順に変更
- 対外ラベルと参照文言を、Byte Helium、Byte Commerce Backend、Byte Portal にそろえるよう更新
- 非 Byte POS 市場向けに、Portal とプラットフォーム関連ドキュメントへ Byte POS に関する注意書きを追加し、Byte Connect を経由する構成を明示
- Front-End Guide と Playbooks の表現を整理し、プラットフォーム責務の言い回しをより一貫させた

### Admin Portal Guide

- Menus 配下に新しい Dayparts カテゴリを追加
- dayparts 概要、作成、店舗別カスタマイズ、価格 / 提供可否、カテゴリ表示設定のガイドを追加
- 店内デバイスと店舗関連ドキュメントに、POS 前提が関係する場面で Byte Connect 依存の説明を追記

### Byte Capabilities

- Byte Helium、Byte Commerce Backend、Byte Portal の新しい概要ページを追加
- Byte Commerce -> Byte Connect -> POS の流れを説明する Byte Connect ページを追加
- プラットフォームメンタルモデルを、静的コードブロックではなくテーマ同期型 iframe 埋め込みへ変更
- プラットフォームレイヤー、用語集、ローンチガイダンス、ドメインページを新しい Byte 命名モデルへ更新

### プロトタイプ整理

- sneak-peek iframe が、廃止予定の `sneak-peek-prototype.html` ではなく `rna-sneak-peek-prototype.html` を利用するよう更新
- 旧 `sneak-peek-prototype.html` を削除し、プロトタイプ資産を整理

## ひとことで言うと

今回のリリースで、ドキュメント全体の意図がかなり明確になりました。Atlas Peek プロトタイプやナビバー改善も引き続き有効ですが、特に大きいのは Byte スタックの説明が分かりやすくなり、Portal 利用者向けの daypart ガイドが増え、非 Byte POS 市場で Byte Connect がどこに入るのかまで含めて整理されたことです。
