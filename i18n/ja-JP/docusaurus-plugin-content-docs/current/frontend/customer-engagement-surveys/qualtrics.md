# Qualtrics

**機能領域:** [カスタマーエンゲージメント調査](./)

Qualtrics は、**KFC Listens** 型のカスタマーエンゲージメント調査で使われるアンケート基盤です。アンケート配信、分岐ロジック、回答収集、レポーティング運用を担い、顧客体験チームや市場運営チームが活用します。

## Qualtrics が担うこと

Atlas のエコシステムでは、Qualtrics は通常次の用途を支えます。

- Web リンクや QR コードから表示されるアンケートフォーム
- チャネル、市場、回答タイプに応じた条件分岐質問
- CSAT、NPS、理由コードなどの満足度指標
- CX レポーティング用の回答エクスポートとダッシュボード
- フォローアップが必要な低評価回答へのアラート

## 代表的な KFC Listens フロー

```text
1. お客様が注文または来店体験を完了する
2. KFC がアンケート招待を共有する（レシート、メール、プッシュ通知、QR コードなど）
3. お客様が Qualtrics のアンケートを開く
4. Qualtrics が評価、コメント、店舗体験の詳細を収集する
5. 市場チームがダッシュボードを確認し、サービスリカバリー案件に対応する
```

## 代表的に収集されるデータ

| データ項目 | 例 |
|---|---|
| 来店または注文参照情報 | 注文番号、店舗 ID、来店日 |
| チャネル | 店内飲食、持ち帰り、デリバリー、アプリ、Web |
| 満足度スコア | 来店または注文全体の評価 |
| 課題カテゴリ | 商品品質、不足品、スピード、スタッフ対応 |
| 自由記述フィードバック | お客様自身のコメント |

## 運用上の考慮点

### アンケート配信タイミング

市場はいつお客様へ招待を送るかを決めます。一般的には、チェックアウト直後、受け渡し後、または店頭で使われるレシートリンク経由などのパターンがあります。

### インセンティブ

プログラムによっては、アンケート完了後に認証コードやオファーを提供することがあります。インセンティブルールは市場レベルで管理され、プロモーションまたは CRM チームとの調整が必要になる場合があります。

### エスカレーション

低評価回答は、カスタマーサポートや CX レビューフローへ送られることがあります。市場は次の点を定義しておく必要があります。

- どのスコア閾値でアラートを作るか
- どのチームがフォローアップを担当するか
- どれくらいの時間内にお客様へ連絡すべきか

## 連携メモ

Qualtrics は Atlas に隣接する仕組みであり、コアの注文フローそのものではありません。アンケート体験は Atlas の注文データを参照できますが、アンケート基盤自体はメインの注文アプリケーションとは別で管理されます。

:::tip
KFC Listens プログラムを開始または変更する場合は、公開前にアンケート文言、配信タイミング、サービスリカバリーの SLA を揃えてください。タイミングがよくてもフォローアップ責任が曖昧だと、有益なインサイトではなく運用ノイズになりやすくなります。
:::
