# カスタマージャーニー

このセクションでは、顧客向けドキュメントを実際の利用行動に沿って整理します。内部機能単位ではなく、お客様が体験の中をどう移動するかを基準に構成しています。

以下の構成は、グローバルで最も一般的なデジタル注文パターンに沿っています。お客様は最初に受け取り条件を決め、その後に商品やお得情報を探し、商品詳細を確認してカスタマイズし、チェックアウトを完了し、最後に注文を追跡します。アカウント、店舗検索、ヘルプ、ブランドコンテンツなどの補助導線は、この主導線の外側に並行して存在します。

このジャーニー内の各接点には専用のサブページがあります。各ページでは、モックアップ、設計意図、業務ルール、市場別の差分を整理できます。

## 主要な注文ジャーニー

| ステップ | ジャーニー段階 | お客様が達成したいこと |
|---|---|---|
| 1 | **注文設定** | 適切なメニューを表示するために、場所、注文モード、店舗、時間を設定する |
| 2 | **商品を探す** | メニュー、カテゴリ、商品入口を閲覧する |
| 3 | **ディールと特典を探す** | プロモーション、バウチャー、ロイヤルティ特典、お得なオファーを見つける |
| 4 | **商品確認とカスタマイズ** | 商品内容を理解し、モディファイアを選び、追加提案を確認してから追加する |
| 5 | **注文と受け取り** | カート確認、プロモ適用、支払い方法選択、注文確定を行う |
| 6 | **注文追跡** | 受け取りまたは配送の状態を受け渡し完了まで確認する |

## この構成が適している理由

1. 最初に条件設定が必要なのは、場所、受け取り方法、時間によって実際に注文できる内容が決まるためです。
2. その後、お客様は商品探索とお得情報探索を行き来しながら候補を比較します。
3. 商品やオファーの候補が見つかってから、詳細確認、カスタマイズ、カート確認、チェックアウトに進みます。
4. 購入後は、行動の中心が商品探索から追跡とサポートへ移ります。
5. アカウント、ヘルプ、店舗検索、ブランド情報も重要ですが、注文前・注文中・注文後のどこでも起こり得るため、補助ジャーニーとして整理するのが適しています。
