# フロントエンド概要

**KFC Atlas** は、デジタル注文のための KFC のグローバルフロントエンドです。オンライン注文、KFCアプリ、外部アグリゲーター経由の注文時に表示される体験がここに含まれます。

このガイドは、**店舗チームとBMU**向けです。お客様にどのような体験が提供されるのか、市場ごとにどう振る舞うのか、またプロダクトの再デプロイなしで何を設定できるのかを理解するために使います。

## フロントエンドで扱う内容

Atlasのフロントエンドは、主に次の4領域で構成されています。

- **カスタマージャーニー** — 検索から注文確定までの一連の流れ
- **注文チャネル** — 店内飲食、デリバリー、持ち帰りが店舗ごとにどう表示・管理されるか
- **カスタマーエンゲージメント調査** — KFC Listens 型のフィードバックプログラムが注文後にどう機能するか
- **市場設定** — 市場ごとにオン・オフできる機能（ロイヤルティ、プロモーション、スケジュール設定など）

## デザインシステム

KFC Atlasのフロントエンドは、**Byte Helium** の上に構築されています。Byte Helium は Yum! のエンタープライズ向けデザインシステム兼フロントエンド基盤であり、共通のデザイントークン、UIコンポーネント、構造パターンを提供します。Atlas はその基盤の上で、KFC ブランドに合わせた見た目と顧客体験を実現しています。

## このガイドを読むべき人

| 役割 | 関連する理由 |
|---|---|
| 店舗運営者 / 市場チーム | お客様の体験と、自分たちで設定できる項目を理解するため |
| BMU / 地域オフィス | 市場レベルの設定項目と、機能の有効化方法を理解するため |
| プリセールス / 新市場オンボーディング担当 | ローンチ前に、標準機能として何が提供されるかを把握するため |

:::info
フロントエンドは、すべての市場で共通のコードベース上に構築されています。市場ごとの差分は個別ビルドではなく、設定で制御されます。
:::
