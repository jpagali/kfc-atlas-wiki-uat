# Contentful概要

Contentful は、KFC Atlas の Web 体験におけるコンテンツ編集レイヤーを支える CMS です。市場チーム、コンテンツ担当者、メニュー担当者、翻訳担当者は、プロダクトの再デプロイやエンジニアリング支援なしで、フロントエンドに表示される内容を管理できます。

KFC のデジタル接点でお客様が*何を見るか*をコントロールするための管理画面と考えると分かりやすいです。どの商品を見せるか、プロモーションをどう表示するか、どんな文言を出すか、ページをどう構成するかを、**Web** と **モバイルアプリ** の両方で管理できます。

## 利用する人

| 役割 | Contentful で行うこと |
|---|---|
| **Content Creator** | ホームページバナー、プロモーション、ページ文言の更新・公開 |
| **Menu Author** | 商品、カテゴリ、バンドル、ディールの管理 |
| **Translator** | 市場内で利用する各言語向けのローカライズ管理 |

## 管理できる内容

| コンテンツ領域 | 説明 |
|---|---|
| **Homepage** | ヒーローバナー、注目商品、キャンペーンタイル |
| **Categories** | メニューカテゴリ構成、ラベル、画像 |
| **Products** | 個別商品の表示名、説明、画像、価格表示補助 |
| **Bundles & Deals** | セット商品、コンボオファー、バンドル構成 |
| **Promotions** | 期間限定プロモーションやキャンペーン訴求 |
| **Cross-Sell in Cart** | カート内のおすすめ商品表示 |
| **Footer** | ナビゲーションリンク、法務文言、関連リンク |
| **SEO** | ページタイトル、メタディスクリプション、canonical URL |
| **Translations** | 上記すべてに対する多言語コンテンツ |

## 公開の流れ

Contentful では、次のような構造化された公開フローを使います。

1. コンテンツ担当者またはメニュー担当者が**下書き**を作成する
2. 社内で**レビュー・承認**を行う
3. **Launch** を使って公開する  
   Contentful のスケジュール公開機能で、複数エントリの公開タイミングをそろえられます

これにより、キャンペーンを事前に準備し、公開時刻にオンライン待機しなくても、狙ったタイミングで反映できます。

## 対応プラットフォーム

Contentful で管理したコンテンツは、Web とモバイルアプリの両方で利用されます。同じ商品説明、プロモーション文言、バナー、カテゴリラベルが Atlas API レイヤーを通じて両プラットフォームに配信されます。

| プラットフォーム | 対応状況 |
|---|---|
| **Web** | ✅ 全コンテンツタイプに対応 |
| **iOS / Android** | ✅ 対応。基本的には同じ API 経由で配信されるが、バナー比率など一部レイアウト依存項目は差が出る場合あり |

> **注記:** プッシュ通知文言やストアメタデータなど、一部のアプリ専用設定は Contentful 外で管理されます。このガイドでは、KFC アプリ体験の中に実際に表示されるコンテンツのみを対象にしています。
