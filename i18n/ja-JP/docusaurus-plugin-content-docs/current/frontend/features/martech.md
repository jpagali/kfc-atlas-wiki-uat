# MarTech

## 機能概要

MarTech は、KFC 体験のあらゆる接点で発生する顧客行動データを収集し、連携し、活用するための Atlas プラットフォームのマーケティング技術基盤です。

この機能は、主に次の 3 つの仕組みで構成されています。

- **Event Tracking** — アプリや Web 上で顧客が何をしたかを記録する
- **mParticle** — 顧客データプラットフォーム（CDP）として全イベントを受け取り、下流システムへルーティングする
- **Braze** — プッシュ通知、アプリ内メッセージ、マーケティング自動化を実行する顧客エンゲージメント基盤

これらを組み合わせることで、各市場はキャンペーンごとにエンジニアリングへ依頼しなくても、パーソナライズされたデータ活用施策を実行できます。

## データの流れ

```text
顧客アクション（アプリ / Web）
        ↓
イベント計測レイヤー
  （構造化イベントを送信）
        ↓
     mParticle
  （データ収集・連携）
        ↓
       Braze
（キャンペーン、Push、In-App）
```

## 構成要素

### Event Tracking
Atlas では、商品閲覧、カート追加、注文完了、プロモーションタップなど、意味のある顧客行動ごとに構造化イベントを発火します。イベントは共通スキーマで設計されているため、市場をまたいで比較・分析しやすくなっています。

### mParticle
mParticle は中心となるハブです。Atlas から送られたイベントを受け取り、正規化・補完したうえで、Braze や分析ツール、その他設定済みの外部連携先へ必要なデータを渡します。

### Braze
Braze は mParticle から受け取った顧客プロフィールとイベントデータを使って、次のような施策を実行します。

- プッシュ通知（注文更新、プロモーション、再来店施策）
- アプリ内メッセージ（パーソナライズバナー、オンボーディング案内）
- マーケティング自動化（顧客行動を起点にしたトリガー配信）
