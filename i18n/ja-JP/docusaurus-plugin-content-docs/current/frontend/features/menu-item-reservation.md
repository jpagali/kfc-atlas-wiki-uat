# メニュー商品の予約

## 機能概要

Menu Item Reservation を使うと、通常販売が始まる前に、特定メニューを**事前予約して確保**できます。在庫が残っていることを期待して来店するのではなく、顧客は受取日と受取時間帯を選び、事前決済を行ったうえで予約を確定できます。

この機能は、需要は読めるが供給に制限がある場面に向いています。たとえば、季節キャンペーン、限定商品の発売、大型イベント、あるいは文化的な意味合いが強いメニュー施策などです。

## なぜ必要か

限定商品や季節限定セットのようなメニューは、店舗がリアルタイムで対応できる処理能力を大きく超える需要を生むことがあります。予約機能がない場合、次の問題が起こりやすくなります。

- 顧客が来店しても商品が売り切れている
- 店舗が予測しづらい当日需要で混乱する
- マーケティングで需要を作れても、注文システム側で受け止めきれない

Menu Item Reservation は、そうした需要を確定済み・スケジュール済みの注文へ変換し、顧客には確実性を、店舗には準備の見通しを提供します。

## 仕組み

予約フローは大きく 4 ステップです。

```text
1. 閲覧       → 顧客が予約可能商品を見つける（商品ページ、ホームバナー、プロモーションなど）
2. 予約       → 受取日と時間帯を選び、数量を確定する
3. 支払い     → 予約枠を確保するため、全額または一部を事前決済する
4. 受け取り   → 顧客が店舗で予約確認情報を提示して商品を受け取る
```

### 予約受付期間
予約対象商品には **reservation window** が設定されます。これは事前予約を受け付ける期間です。この期間外では、商品は通常商品として表示されるか、完全に非表示になります。

### 時間帯
店舗ごとに、1日あたりの**受取時間帯**を設定できます。各時間帯には上限数があり、過剰予約を防ぎます。ある時間帯が満枠になると、顧客には次に空いている時間帯が提示されます。

### 支払い
予約確定には事前決済が必要です。これにより no-show を減らし、店舗は確定済み注文だけを準備できます。キャンセルや返金ポリシーは市場ごとに設定されます。

## 設定項目

Menu Item Reservation は、**Contentful**（主に Web 向け設定）と市場設定レイヤーを通じて商品単位で有効化されます。運用チームは次の項目を設定できます。

| 設定項目 | 内容 |
|---|---|
| **Reservation window** | 予約受付の開始日と終了日 |
| **Pickup dates** | 予約受取を受け付ける日付 |
| **Time slots** | 利用可能な受取時間帯と各時間帯の上限数 |
| **Deposit / full payment** | 予約時に全額決済か一部入金か |
| **Cancellation policy** | キャンセル可否と条件 |
| **Max quantity per order** | 1顧客が予約できる最大数量 |
