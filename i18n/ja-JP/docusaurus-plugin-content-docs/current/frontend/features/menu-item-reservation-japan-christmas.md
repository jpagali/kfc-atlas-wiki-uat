# 日本 — クリスマス注文

**Feature:** [Menu Item Reservation](./menu-item-reservation) · **Market:** 🇯🇵 Japan

---

## 背景

KFC Japan のクリスマスキャンペーンは、世界でも象徴的なファストフード文化のひとつです。1974年以降、KFC Japan は *"Kurisumasu ni wa Kentakkii"*（ケンタッキーフォークリスマス）キャンペーンを展開しており、日本の多くの家庭にとって KFC がクリスマスの定番になっています。

12月24日と25日には、KFC Japan は通常日の約10倍にあたる **約360万人** の顧客に対応します。この非常に高い需要を安定して処理するため、市場では長年にわたり**事前予約システム**を運用しており、顧客は数週間から数か月前にクリスマスメニューを確保できます。

Atlas における Menu Item Reservation は、この事前予約体験をプラットフォーム標準機能として実現するもので、従来の POS ベース予約運用を、アプリと Web 上のシームレスな体験へ置き換えます。

## 日本での予約フロー

### 1. キャンペーン開始（10月〜11月）

クリスマス予約は、通常 12月24日の **6〜8週間前** に開始されます。この期間中は次のような導線が展開されます。

- KFC Japan のアプリや Web で、ホームバナー、プッシュ通知、専用ランディングページなどを通じてクリスマス施策を大きく訴求する
- Party Barrel の商品ページでは、通常の「カートに追加」ではなく **「今すぐ予約」** CTA を表示する
- 受け取り可能日（通常は 12月23日〜25日）と時間帯を案内する

### 2. 顧客が予約する

顧客は次の流れで予約を完了します。

1. **セットを選ぶ** — Standard Barrel や Premium Barrel などから選択
2. **店舗を選ぶ** — 受け取り希望店舗を指定
3. **受け取り日時を選ぶ** — 12月23日、24日、25日の中から特定時間帯を選択
4. **数量を選ぶ** — 1回の注文で予約できる上限数の範囲で指定
5. **内容を確認して支払う** — 予約確定のため、事前決済を行う

### 3. 予約確認

支払い完了後、顧客には次の情報が案内されます。

- 予約番号を含むアプリ内確認画面
- 予約完了を知らせるプッシュ通知
- 店舗住所、受取時間、来店時に提示する内容を含む確認メール

### 4. 受け取り（12月23日〜25日）

顧客は予約した時間帯に対象店舗へ来店し、アプリまたは印刷した確認情報を提示して商品を受け取ります。注文は予約数量に基づいて事前に準備されるため、当日の待ち時間や不確実性を大きく減らせます。

## なぜ事前決済が必要か

日本のクリスマス予約では、デポジットではなく**全額事前決済**を採用しています。これはこのユースケースに適した設計です。

| 理由 | 内容 |
|---|---|
| **無断キャンセルを抑えられる** | Party Barrel は予約数量に合わせて準備するため、来店しないとロスが大きい |
| **在庫・準備量を正確に見積もれる** | 店舗ごと、時間帯ごとに必要数を把握できる |
| **ピーク日の混雑を減らせる** | 事前決済済み顧客は当日オーダーより受け渡しを簡素化しやすい |
| **限られた在庫を守れる** | 支払いなしの仮予約だと、高需要時間帯が実需以外で埋まる可能性がある |

## 需要とキャパシティ管理

Atlas では、店舗ごとに設定された**時間帯別の上限数**で需要を制御します。

- 各店舗は、時間帯ごとの提供可能数を設定する
- 上限に達した時間帯は自動的に満枠として扱われ、選択肢から外れる
- 顧客は次に空いている時間帯、または別店舗へ案内される

これにより、店舗単位の過剰予約を防ぎながら、受取期間全体に需要を分散できます。

## 設定サマリー

| 項目 | 日本のクリスマス設定 |
|---|---|
| **予約受付開始** | 12月24日の約 6〜8週間前 |
| **受取日** | 12月23日、24日、25日 |
| **時間枠** | 営業時間内の時間帯単位 |
| **時間帯ごとの上限** | 各店舗の厨房キャパシティに応じて設定 |
| **支払い方法** | 全額事前決済 |
| **キャンセル** | 市場設定による（通常は受取1週間前以降を制限） |
| **1注文あたり上限数** | 2〜3セット程度（市場設定） |

## 他市場への示唆

Japan の Christmas campaign は、Menu Item Reservation が通常注文を超えて、**文化的に重要で大量需要が発生するイベント**にも使えることを示しています。ほかの市場でも参考になるポイントは次の通りです。

- 需要分散のため、十分早く予約受付を開始する
- 高単価商品や限定商品の場合、全額事前決済は no-show 抑制に有効
- 上限管理は全体数ではなく、店舗ごとの時間帯単位で行う
- ピーク需要が始まる前に、プッシュ通知やホーム訴求で予約認知を高める
- アプリ内、プッシュ、メールなど複数導線で確認情報を届ける
