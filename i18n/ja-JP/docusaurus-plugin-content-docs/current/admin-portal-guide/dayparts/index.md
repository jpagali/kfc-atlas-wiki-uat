# Dayparts / デイパート

## デイパートとは

クイックサービスレストラン業界では、**デイパート** とは、来店傾向や提供メニューを定義する 1 日の時間帯区分を指します。QSR では、時間帯ごとの顧客行動に合わせて運用を最適化し、プロモーションを調整し、売上を高めるためにデイパートを活用します。

## よくあるデイパートの利用例

- **Breakfast:** `8:00 AM - 10:30 AM`（朝の時間帯だけ朝食メニューを提供）
- **Happy Hour:** `3:00 PM - 5:00 PM`（閑散時間帯の割引で集客を強化）
- **Late Night:** `10:00 PM - Close`（限定メニューや深夜価格で提供）

## Byte プラットフォームでできること

1. **商品、バリアント、バンドル単位のデイパート別提供可否設定**

   朝食メニューのような時間帯限定提供を実現できます。

2. **バリアントとバンドルのデイパート別価格設定**

   時間帯によって価格を変えることができます。

3. **オケージョン別価格設定**

   バリアントとバンドルを、オケージョンごとに異なる価格で設定できます。

4. **カテゴリのデイパート別表示制御**

   特定のカテゴリを非表示にしたり、表示優先度を下げたりできます。

5. **パッチによるデイパート価格変更**

   カタログ上で設定されたデイパート価格や提供可否は、パッチでも設定できます。

6. **組織レベルと店舗レベルの両方でデイパート時間枠とオケージョンを設定**

   デイパート自体は組織レベルでのみ作成しますが、営業時間や対応オケージョンは各店舗ごとに細かく調整できます。
