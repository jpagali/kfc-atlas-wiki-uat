# Menú Reserva de artículos

## Lo que hace

Menú Reserva de artículos permite a los clientes **pre-orden y garantizar elementos de menú específicos** antes de que estén disponibles para el pedido estándar. En lugar de esperar que un producto esté disponible cuando lleguen, los clientes se reservan con antelación — seleccionando una fecha de recogida, la ranura del tiempo y completando el pago por adelantado.

Esta característica está diseñada para situaciones en las que la demanda es predecible pero la oferta es limitada: campañas estacionales, lanzamientos exclusivos de productos, eventos de alto tráfico o momentos culturales donde una comida específica tiene un significado elevado.

## Por qué Existe

Algunos elementos de menú —especialmente productos de edición limitada o conjuntos de comidas culturalmente significativos— generan demanda que excede con creces la capacidad de cumplimiento de un restaurante en tiempo real. Sin sistema de reservas:

- Los clientes llegan para encontrar los artículos vendidos
- Los restaurantes están abrumados por una demanda impredecible de entrada
- Las campañas de marketing impulsan la intención de que el sistema de pedidos no pueda capturar

Menu Artículo Reserva convertidos que demandan en pedidos confirmados, programados — dando a los clientes una garantía y dando a los restaurantes la visibilidad para prepararse con precisión.

## Cómo funciona

El flujo de reserva tiene cuatro pasos:

```
1. Browse        → Customer discovers a reservable item (product page, homepage banner, promo)
2. Reserve       → Customer selects pickup date and time slot, confirms quantity
3. Pay           → Full or partial payment is taken upfront to hold the reservation
4. Collect       → Customer arrives at the restaurant, presents reservation confirmation
```

### Ventana de reserva
Cada elemento reservable tiene una ventana de reserva configurada **** — el período durante el cual se abren los pedidos previos. Fuera de esta ventana, el artículo aparece como un producto de menú estándar (si está disponible) o se oculta por completo.

### Ranuras del tiempo
Los restaurantes están configurados con ranuras de tiempo **pickup** por día. Cada ranura tiene un límite de capacidad, evitando la sobre-booking. Una vez que la ranura está llena, los clientes se ofrecen la próxima vez disponible.

### Pago
Se requiere el pago anticipado para confirmar una reserva. Esto reduce la no presentación y asegura que el restaurante sólo prepara órdenes confirmadas. Las políticas de cancelación y reembolso están configuradas por mercado.

## Configuración

Menú Reserva de artículos está habilitada a nivel de producto mediante **Contenido** (para web) y la capa de configuración del mercado. Los operadores pueden configurar:

| Ajuste | Descripción |
|---|---|
| ** ventana de reserva** | Fecha abierta y fecha de cierre para pre-ordenes |
| # Fechas de presentación # | Para qué fechas se aceptan reservas |
| **Tiempos de tiempo** | Ventanas disponibles de recogida y capacidad por lotes |
| **Depósito / pago completo** | Si el pago completo o un depósito se carga en la reserva |
| **Política de selección** | Si los clientes pueden cancelar y en qué condiciones |
| ** Cantidad máxima por orden** | Limite cuántos clientes pueden reservar |
