Menüs

Dieser Abschnitt umfasst alle Menüverwaltungsaufgaben im Byte Commerce Admin Portal, einschließlich der Erstellung von Menüs, der Organisation in Kategorien, der Konfigurierung gezielter Überschreitungen mit Patches und der Verwaltung von Preisstrukturen.

Menüs​

  • Menü erstellen— Erstellen Sie eine neue Menüstruktur für Speicher und Kanäle
  • Menü bearbeiten— Aktualisieren der Konfiguration eines vorhandenen Menüs
  • Menü kopieren— Duplizieren Sie ein Menü als Ausgangspunkt
  • Ein Menü zuordnen— Verlinken Sie ein Menü zu bestimmten Speichern und Kanälen
  • Menü veröffentlichen— Schieben Sie ein Menü, um die Bestellkanäle zu leben
  • Löschen eines Menüs— Entfernen Sie ein Menü aus dem System

Kategorien​

  • Eine Kategorie erstellen— Fügen Sie ein Gruppierlabel hinzu, um Produkte innerhalb eines Menüs zu organisieren
  • Kategorie bearbeiten— Name oder Konfiguration einer Kategorie aktualisieren
  • Kategorie kopieren— Duplizieren einer Kategorie mit ihrer Konfiguration
  • Kategorie löschen— Entfernen Sie eine Kategorie aus einem Menü
  • Metafields zu einer Kategorie hinzufügen— Benutzerdefinierte Daten an eine Kategorie für Integrationen anhängen

Aufnäher​

  • Einen Patch erstellen— Erstellen Sie ein gezieltes Überschreiben für Menüänderungen
  • Einen Patch bearbeiten— Aktualisieren der Operationen oder Elemente eines Patches
  • Einen Patch kopieren— Vervielfältigen eines Patches als Ausgangspunkt
  • Löschen eines Patches— Entfernen Sie einen Patch aus dem System
  • Einen Patch zuordnen (Zu Patch hinzufügen)— Fügen Sie einen Patch in die aktive Liste eines Speichers hinzu
  • Zuordnen eines Patches (Ersetzen vorhandene Liste)— Ersetzen Sie die gesamte Patchliste eines Speichers

Name der Preise​

  • Einen Namenspreis erstellen— Definieren Sie ein wiederverwendbares Preislabel für mehrere Produkte
  • Bearbeiten Name des Preises— Aktualisieren des Wertes eines bereits genannten Preises
  • Gelöschter Name Preis— Entfernen Sie den Namen des Systems

Verwandte Domains​

  • Tagteile— Verwalten org-Level Dayparts, Überschreitungen auf Lagerebene, zeitbasierte Preise, Verfügbarkeit und Sichtbarkeit der Kategorie
