Admin Portal Guide

This guide covers all tasks available in the Byte Commerce Admin Portal. Select a section below to get started.

Source Acknowledgement

This Admin Portal Guide was built from the Byte Team's Prototype User Guide and adapted for Atlas Wiki.

Open the Byte Team Prototype User Guide in Figma

  • YumKnowledge Base: Admin Portal User Guide
Byte POS Caveat

Admin Portal tasks that reference POS, in-store devices, or store-to-POS communication assume the market is using Byte POS.

If a market is not using Byte POS, Byte Connect must be onboarded as part of the Byte Commerce bundle so the integration path becomes Byte Commerce -> Byte Connect -> POS.

Stores​

Manage restaurant locations, menus, and store settings.

Menus​

Create and organize product menus and categories for each store.

Dayparts​

Manage time-based menu behavior, including org-level daypart setup, store-level overrides, pricing, availability, and category visibility.

Products​

Define individual menu items, ingredients, pricing, and variants.

Bundles​

Build meal deals and combos by grouping products together.

Promotions​

Create and manage promotional offers and discount campaigns.

Store Groups​

Organize multiple stores and manage group-level settings and promotions.

Agents​

Create and manage user accounts with specific portal access and permissions.

Customer Support​

Search orders and customers, issue refunds, and assist with account issues.

In-Store Devices​

Manage POS terminals and kiosks, including device setup and authentication.
