# Customizing Dayparts at the Store-level

## Summary

Dayparts can only be created at the org-level, but occasions and timeframes can be customized at each store.

**Screenshot placeholder:** Put screenshot showing the store-level dayparts inheritance diagram with organization defaults and store overrides here.

## How to customize dayparts at the store-level in Byte Portal

**Step 1:** Starting from the dashboard, click **Stores** in the sidebar.

**Step 2:** Select a specific store.

**Step 3:** Click the **Dayparts** tab in the top-right navigation bar.

**Screenshot placeholder:** Put screenshot showing a store details page with the Dayparts tab in the top-right navigation bar here.

**Step 4:** On the daypart you wish to edit, click the **kebab icon** in the rightmost cell of the table and click **Edit**.

**Step 5:** Make any changes you wish: edit existing timeframes, add timeframes, or delete timeframes.

**Screenshot placeholder:** Put screenshot showing the store-level daypart edit view with editable occasions and timeframes here.

**Step 6:** Save the store-level configuration.

The **Status** column for this daypart will now display **Modified**, indicating that its settings are now different than the organization-level defaults.

**Screenshot placeholder:** Put screenshot showing the store Dayparts table with the Status column displaying Modified here.

:::tip
By default, all stores inherit the same daypart settings from the organization. Customize only the stores that truly need different occasions or timeframes.
:::

:::note
Dayparts can only be defined at the org-level, but a daypart’s supported occasions and timeframes can be customized at each store.
:::

## How to reset a daypart back to org-level settings

**Step 1:** Starting from the dashboard, click **Stores** in the sidebar.

**Step 2:** Select a specific store.

**Step 3:** Click the **Dayparts** tab in the top-right navigation bar.

**Step 4:** On the daypart you wish to edit, click the **kebab icon** in the rightmost cell of the table and click **Edit**.

**Step 5:** Click **Reset to Default**.

**Screenshot placeholder:** Put screenshot showing the store-level daypart editor with the Reset to Default action here.

**Step 6:** Click **Save**.

The **Status** column for this daypart will now display **Default**, indicating that its settings inherit from the organization.
