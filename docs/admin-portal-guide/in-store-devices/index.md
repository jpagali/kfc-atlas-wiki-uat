# In-Store Devices

This section covers management of POS (point-of-sale) terminals and kiosks in KFC restaurants, including device setup, status, and authentication.

:::note Byte POS Caveat
The in-store device guides below are written for markets using **Byte POS**.

If a market is not using Byte POS, **Byte Connect** must be included in the Byte Commerce onboarding bundle, and teams should understand the order path as **Byte Commerce -> Byte Connect -> POS** rather than assuming direct communication from Byte Commerce to the market POS.
:::

## Topics

- [Generate One-Time Password](/docs/admin-portal-guide/in-store-devices/generate-one-time-password/) — Create a temporary authentication code for device setup or re-authentication.
- [View In-Store Device Details](/docs/admin-portal-guide/in-store-devices/view-in-store-device-details/) — Display full details of a device including name, type, status, and menu publish history.
- [Deactivate In-Store](/docs/admin-portal-guide/in-store-devices/deactivate-in-store/) — Disable a device from receiving menu updates or processing orders.

Need the background context? Read [Byte Connect](/docs/byte-capabilities/enablement/byte-connect).
