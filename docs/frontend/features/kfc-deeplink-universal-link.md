# KFC Deeplink | Universal Link

## What It Does

The KFC Universal Link feature lets any supported KFC URL open directly inside the KFC mobile app — taking the user straight to the relevant in-app screen rather than a browser page.

When the **app is installed**, supported links route users directly to the matching screen (e.g., a product page, a promo, or a restaurant landing). When the **app is not installed**, the same link falls back gracefully to the web experience.

The result is a seamless, frictionless path from any touchpoint — a marketing email, a QR code in-restaurant, a social ad, a push notification — straight into the right place in the app.

## Why It Matters

Without Universal Links, every external link to KFC content opens in a browser, even when the customer has the app installed. That creates unnecessary friction — and lost conversion opportunities.

Universal Links close that gap. The URL stays the same. The destination adapts to what the customer has installed.

## Scope

| Area | Detail |
|---|---|
| **Supported domains** | KFC-owned URLs configured per market |
| **Routing** | Path-based — each URL path maps to an in-app destination |
| **App installed** | User is taken directly to the in-app screen |
| **App not installed** | User is sent to the equivalent web experience |
| **Platforms** | iOS, Android |

## Entry Points

Universal Links are typically triggered from:

- Marketing emails and SMS campaigns
- QR codes (in-restaurant, packaging, outdoor)
- Social media ads and organic posts
- Push notifications
- Web-to-app handoff flows

## Platforms

| Platform | Support |
|---|---|
| iOS | ✅ (Apple Universal Links) |
| Android | ✅ (Android App Links) |
| Web | ✅ (fallback destination) |
