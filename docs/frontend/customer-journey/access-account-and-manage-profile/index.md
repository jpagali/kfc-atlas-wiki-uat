# Access Account and Manage Profile

## How This Journey Works

<div className="journey-gif-grid">

<div>

### A. Signed-In User

![Signed-in account-aware destination](/img/customer-journey/gif-library/signed-in/S01-signed-in-home.gif)

**What this shows**

- Shows the account-aware destination after authentication succeeds.
- Confirms that saved context, rewards access, and personalization are available after sign-in.
- Provides the signed-in baseline for account and profile-related journeys.

</div>

<div>

### B. Guest User

![Guest account gate journey](/img/customer-journey/gif-library/guest/G15-guest-more-account-gate.gif)

**What this shows**

- Shows where account-specific tools are gated for unauthenticated customers.
- Makes clear that profile, saved details, and account utilities require sign-in.
- Keeps general browsing available while protecting account-specific actions.

</div>

</div>

**Key difference:** Signed-in customers can access account-aware tools and saved context. Guests can browse and start intent, but authentication is required before profile-specific actions are available.

This supporting journey covers account entry, authentication, profile management, and history-based utilities used before, during, or after ordering.
