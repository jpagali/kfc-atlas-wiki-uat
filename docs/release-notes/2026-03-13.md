# Release 2026.03.13

Released on **March 13, 2026**.

This release focused on turning the Atlas Wiki into a more complete, navigable product. The biggest improvements were around publishing, customer journey coverage, and making the front-end guide feel more structured and easier to browse.

## Highlights

- **GitHub Pages deployment was set up and published**
  The Atlas Wiki now has a clearer deployment path and a published demo snapshot, making it easier to review and share outside local development.

- **Search and routing were stabilized for Pages**
  Search routing and embedded diagram behavior were cleaned up so the hosted experience is more reliable.

- **Customer Journey documentation became a real documentation tree**
  Instead of a single overview page, the customer journey is now broken into step-by-step sections with individual subpages for the main ordering flow and supporting journeys.

- **Customer Engagement Surveys were added to the front-end guide**
  A dedicated survey area now exists, including Qualtrics coverage, so post-order feedback behavior is easier to locate and explain.

- **Japanese navigation support was expanded**
  The site gained additional Japanese customer journey documentation and navigation translation coverage.

## What Changed

### Front-end Guide

- Added a structured **Customer Journey** section with subpages for:
  - Setup Order
  - Find Products
  - Find Deals and Rewards
  - Preview Product and Customize
  - Order and Get
  - Track Order
- Added **Supporting Journeys** for:
  - Access Account and Manage Profile
  - Get Help
  - Find a Store
  - Learn About KFC
- Added **Customer Engagement Surveys** documentation, including a Qualtrics page
- Updated overview and site copy to reflect the newer front-end structure

### Platform and Site Experience

- Set up GitHub Pages deployment
- Published a current demo snapshot
- Fixed Atlas chat links and diagram embeds
- Improved Pages search routing

### Localization

- Added Japanese customer journey docs and navigation translations

## In Plain English

This was the release that made the site feel more like a usable product and less like a loose collection of pages. Navigation is stronger, deployment is clearer, and the customer-facing documentation now has a much better structure for future content.
