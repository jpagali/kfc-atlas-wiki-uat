# Release 2026.03.17

Released on **March 17, 2026**.

This release adds a real production light/dark theme to the Atlas Wiki, with a dark default and a site-wide light mode that applies consistently across the homepage, docs, `What's New`, and the contribution studio. The work also adds a dedicated theme toggle in desktop and mobile navigation, improves dark-mode alert readability, sharpens the homepage hero, makes the Atlas Peek prototype feel more like a prestige loyalty product, and tightens desktop split-screen navbar behavior so the shell stays intentional instead of collapsing awkwardly.

## Highlights

- **The Atlas Wiki now supports a real light/dark mode**
  The site now defaults to dark mode, and users can switch to a true light mode that updates the overall experience instead of leaving major surfaces behind in the previous theme.

- **Theme controls are available on both desktop and mobile**
  A new theme toggle was added to the desktop navbar, and the mobile toggle now lives inside the hamburger menu below the language switcher.

- **The contribution studio now follows the selected theme**
  The publishing studio is no longer visually isolated from the rest of the site and now respects the same site-wide light/dark preference.

- **Dark-mode doc alerts are easier to read**
  Admonitions such as info, tip, warning, and danger callouts now use dark-mode-safe foreground and background combinations so instructional text remains readable.

- **Homepage section-card buttons now respond to theme intentionally**
  The homepage guide/playbook card CTAs now render as KFC red in light mode and white in dark mode.

- **The homepage hero is now sharper and more brand-forward**
  The top hero now leads with `Atlas. Done right.` and a more operational supporting line while keeping the same cinematic background, premium layout, search, and CTA structure.

- **Atlas Peek now includes a prestige-style loyalty tier system**
  The prototype home, rewards, account, cart, checkout, and confirmation surfaces now reflect a `Gold` loyalty state with next-tier progress, premium perk messaging, and stronger status-forward visual treatment.
- **Atlas Peek now supports both App View and Web View**
  Reviewers can now switch between a mobile app-style shell and a website-style shell inside the prototype, making it easier to compare the same ordering flow in both contexts.
- **Atlas Wiki chrome and design-system docs were tightened**
  The main site branding now uses `Atlas Wiki`, `Atlas Peek` has a dedicated navbar CTA, and the Byte Helium architecture diagram now responds more cleanly on smaller screens.

- **Navbar dropdown layering was corrected around Atlas Peek**
  The desktop language dropdown now renders above the sneak-peek review surface instead of getting clipped by nearby layout containers.

## What Changed

### Theme Infrastructure

- Re-enabled functional Docusaurus color-mode behavior while keeping a custom branded toggle UI
- Set the site default to dark mode
- Added a reusable custom theme toggle component
- Added the theme toggle to the desktop navbar
- Added the theme toggle to the mobile hamburger menu below the language toggle
- Kept theme changes client-side without requiring page reloads

### Atlas Peek Prototype

- Refined the embedded Atlas Peek ordering prototype into a more realistic mobile ordering journey
- Added a `Web View` alongside the existing app-style view so the same prototype can be reviewed in both shells
- Added a cleaner `Start My Order` setup flow with delivery and pickup branching
- Added a browse-only menu state so reviewers can inspect the experience before unlocking pricing and cart actions
- Added a mock delivery location flow with typed address entry, suggestion selection, destination confirmation, and automatic store assignment
- Added lightweight change actions so reviewers can switch between store and address setup without losing the overall app feel
- Added a more realistic post-order delivery tracking view with a mock map, rider status, ETA, and destination summary
- Added an embedded immersive/fullscreen mode so the prototype can expand cleanly inside the Atlas Wiki page without losing review context
- Synced the embedded prototype theme with the site-wide light/dark selection
- Removed the duplicate page-level language control so locale switching now happens inside the prototype itself
- Hid Atlas Chat on `/sneak-peek` so the review surface stays focused on the prototype itself
- Improved the prototype mobile chrome by repositioning controls near the Dynamic Island area, adding fullscreen behavior, and tightening small-screen responsiveness
- Reworked the menu hierarchy so Deals & Rewards stays visible without blocking search
- Replaced placeholder menu art with real food photography for the major menu items
- Added a prestige-style loyalty tier layer with `Gold` status, `Platinum` progress, unlock messaging, and rank-aware account/rewards surfaces
- Updated the prototype home welcome state so it shows the current rank instead of a generic welcome-only card
- Replaced the home-screen `Featured Today` header treatment with a compact KFC logo treatment that is better suited for screenshots and recordings

### Global Visual System

- Reworked shared shell tokens for light and dark mode in the global stylesheet
- Removed the prior dark-mode override that effectively forced the app back into a light-only presentation
- Updated navbar, sidebar, footer, search surfaces, and shared chrome to follow the selected theme more consistently

### Atlas Wiki Chrome

- Renamed the primary site chrome from `Atlas Knowledge Center` to `Atlas Wiki`
- Added a dedicated `Atlas Peek` CTA to the desktop navbar
- Tightened desktop navbar spacing so the shorter brand and CTA fit without wrapping
- Added intermediate desktop breakpoints so split-screen widths compress and hide lower-priority navbar controls before the header drops into awkward multi-line wrapping
- Improved the mobile navbar shell with inline search, a clearer sidebar header, a direct Home shortcut, and theme-toggle styling that stays compact
- Improved light-mode visibility for mobile hamburger-menu CTA surfaces so Home and close actions remain easy to spot
- Fixed desktop navbar overflow layering so the locale dropdown can open cleanly above the Atlas Peek surface

### Design System Docs

- Made the Byte Helium design-system architecture diagram more responsive on tablets and mobile
- Replaced several inline sizing rules with reusable classes so the diagram is easier to maintain
- Preserved the same architectural content while making the foundations and market rows wrap more cleanly on small screens

### Homepage and What's New

- Updated homepage branded surfaces to follow shared theme tokens while preserving the current structure and cinematic composition
- Replaced the homepage hero headline with `Atlas. Done right.` and updated the supporting copy to emphasize launch, operations, and scale
- Updated `What's New` cards, labels, and featured areas to respond to the selected theme
- Changed homepage section-card CTA buttons so they are:
  - KFC red in light mode
  - white in dark mode

### Contribution Studio

- Brought the contribution/publishing studio onto the same theme system as the rest of the site
- Updated studio panels, editor surfaces, preview areas, toolbars, and modal overlays to follow the selected light/dark mode

### Documentation Readability

- Improved admonition styling in dark mode for:
  - info
  - note-style secondary alerts
  - tip/success
  - warning
  - danger
- Ensured alert icons, links, headings, inline code, and body text inherit readable contrast in dark mode

## In Plain English

This release makes theme switching feel real instead of cosmetic. The Atlas Wiki now starts dark, can be switched fully into a warm light mode, keeps that preference across the site, and avoids the awkward “some pages changed, some pages did not” behavior that was showing up during review.
