Release 2026.03.15

Released on March 15, 2026.

This release introduced a dedicated What's New experience for the Atlas Wiki and connected it to the main site chrome so launch communication is easier to discover. The work also ensured the new entry points and page experience behave correctly for both English and localized users.

Highlights​

  • A new What's New page is now live The Atlas Wiki now has a dedicated launch page that surfaces featured releases, new features, recent improvements, and upcoming work in a more structured product-marketing format.

  • What's New is now part of the main navigation A new top-nav entry was added beside Playbooks, while About was removed from the top navigation but kept available in the footer.

  • The footer now includes a twin launch card The existing The 12th Secret footer callout is now paired with a What's New card so launch updates have a strong desktop entry point.

  • Mobile behavior stays intentionally lightweight On mobile, the What's New page remains discoverable through the hamburger navigation without adding extra footer clutter.

  • localized routing and page content are supported The new page and release-note coverage were localized so the feature works in both en-US and retired locale.

What Changed​

Navigation and Entry Points​

  • Added a What's New item to the top navbar
  • Removed About from the top navbar
  • Kept About Atlas Wiki reachable from the footer
  • Added a desktop footer twin-card entry point for What's New
  • Kept mobile access focused on the hamburger navigation

What's New Experience​

  • Added a new /whats-new page in the app
  • Styled the page as a dark KFC-branded launch hub
  • Structured the page around:
    • roadmap snapshot
    • featured launch
    • new features
    • recent improvements
    • coming soon
  • Added stronger visual treatment for the featured launch area

Localization​

  • Added localized navbar copy for What's New
  • Added localized page content for /retired locale/whats-new
  • Added localized release-notes coverage so launch communication remains navigable in both locales
  • Fixed an Atlas Chat quick-link routing bug that could send localized users to a page-not-found route when opening linked docs from the chat suggestions
  • Updated the Atlas Peek prototype so both shells keep an in-context language toggle: the mobile app shell uses the top toolbar, while the simulated web shell shows the switch inside the web nav

Layout Polish​

  • Adjusted footer spacing so the new twin-card layout does not overly constrain the footer columns
  • Improved localized footer readability by giving link text and CTA copy more breathing room

In Plain English​

This release turned launch communication into a real product surface instead of a hidden changelog. People can now find What's New from the main navigation and footer on desktop, still reach it cleanly on mobile, and use it in both English and localized without breaking localization behavior.
