# Release 2026.03.19

Released on **March 19, 2026**.

This release cleans up Atlas Wiki terminology and navigation, expands the Byte documentation set with daypart and platform-reference content, adds a theme-aware platform hierarchy embed, and carries forward the Atlas Peek prototype and desktop navbar fixes from the earlier March 19 build.

## Highlights

- **Atlas Peek prototype now renders correctly in light mode**
  The PICKUP/ASAP chips, topbar, navigation, and journey-shell bar now display with proper light-mode colors instead of defaulting to dark-mode styling.

- **Duplicate sidebar category names resolved**
  The Admin Portal Guide sidebar had nested categories with identical labels (Stores, Menus, Products, etc.). These have been renamed to "Tasks" subcategories to prevent translation conflicts and improve clarity.

- **Navbar now responds to browser zoom levels**
  The search bar in the navbar now shrinks progressively as the viewport narrows (from 280px down to hiding completely), ensuring navigation links remain visible at various zoom levels.

- **Contribute button removed from desktop navbar**
  The Contribute button was removed from the desktop navbar to reduce clutter. Users can still access the contribution studio via the footer CTA.

- **Search placeholder updated**
  The navbar search bar now uses "What's on your mind?" as the placeholder text for a more conversational feel.

- **Hero search bar width adjusted**
  The hero search bar max-width was reduced from 720px to 600px for better visual balance.

- **Byte Capabilities credits updated**
  The credits page now reflects "The Byte Capabilities" and includes a link to the original Confluence documentation.

- **Byte terminology and navigation were tightened across the docs**
  Navigation and page copy now align around `Byte Helium`, `Byte Commerce Backend`, and `Byte Portal`, making the platform structure easier to understand at a glance.

- **The Byte Portal Guide now covers dayparts end to end**
  The Menus section now includes a dedicated dayparts overview plus task guides for creating dayparts, customizing them per store, and configuring pricing, availability, and category visibility.

- **Byte Capabilities now has clearer platform entry points**
  The section now breaks out dedicated overview pages for Byte Helium, Byte Commerce Backend, and Byte Portal, and adds a new Byte Connect explainer for non-Byte POS markets.

- **The platform mental model now uses a theme-synced hierarchy embed**
  The old static hierarchy block was replaced with an embeddable diagram that follows the site theme and reflects the Byte Connect to market POS flow more clearly.

## In Plain English

This release makes the docs feel much more intentional. The Atlas Peek prototype and navbar polish are still here, but the bigger gain is that the wiki now explains the Byte stack more clearly, adds daypart guidance for portal users, and documents where Byte Connect fits when a market is not using Byte POS.
