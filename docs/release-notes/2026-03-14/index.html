Release 2026.03.14

Released on March 14, 2026.

This release focused on expanding contribution workflows while restoring stability in the shared site navigation. The headline addition is a new guided contribution studio for submitting Atlas Wiki content, paired with a reset of locale navigation back to the safer built-in Docusaurus behavior.

Highlights​

  • A new contribution studio is now available Contributors can draft Atlas Wiki content in a branded editor, preview the article, download Markdown, and open a publishing-request email without needing repository or pull request access.

  • Contribute CTAs are now visible in the main site chrome A new Contribute CTA appears in the navbar, and a footer-side callout introduces the contribution flow as The 12th Secret.

  • localized coverage now includes the contribution studio The studio experience now has localized-facing labels and copy so the new workflow is available in both site locales.

  • Locale switching was stabilized by reverting to the framework default After custom locale-switcher experiments proved brittle, navigation was reset to Docusaurus' built-in locale dropdown so language behavior is handled by the platform instead of custom client-side path logic.

  • The new authoring experience was kept lightweight The contribution flow was implemented without introducing a heavy editor dependency or a raw-media upload backend.

What Changed​

Contribution Workflow​

  • Added a new /contribute/ submission page with:
    • guided metadata capture
    • template starters
    • a WYSIWYG authoring surface
    • collapsible preview
    • Markdown export
    • publishing-request email drafting
  • Added linked-image preview behavior with an attachment reminder when an image URL fails to render
  • Added localized copy coverage for the contribution studio
  • Added prominent contribution entry points in the navbar and footer

Site Experience​

  • Added a footer callout labeled The 12th Secret
  • Styled the contribution studio as a dark KFC-branded publishing surface
  • Kept the main reading experience lightweight by isolating contribution tooling to its own page

Localization and Stability​

  • Removed the custom locale-switcher implementation after it proved unreliable
  • Restored Docusaurus' built-in locale dropdown for safer language navigation
  • Rebuilt the static output after the locale-navigation reset

Developer Experience​

  • Kept the implementation lightweight by avoiding a heavy editor package
  • Preserved the existing docs build flow while adding the new contribution route
  • Continued using single-port built preview as the safest way to validate cross-locale browsing locally

In Plain English​

This release added a polished new way for people to contribute content to the Atlas Wiki without touching git, while also backing away from risky custom locale-routing behavior in favor of the framework’s safer default navigation. The result is a more welcoming contribution experience with a more stable localization baseline.
