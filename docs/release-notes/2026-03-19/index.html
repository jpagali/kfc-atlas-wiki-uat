Release 2026.03.19

Released on March 19, 2026.

This release cleans up Atlas Wiki terminology and navigation, expands the Byte documentation set with daypart and platform-reference content, adds a theme-aware platform hierarchy embed, and carries forward the Atlas Peek prototype and desktop navbar fixes from the earlier March 19 build.

Highlights​

  • Atlas Peek prototype now renders correctly in light mode The PICKUP/ASAP chips, topbar, navigation, and journey-shell bar now display with proper light-mode colors instead of defaulting to dark-mode styling.

  • Duplicate sidebar category names resolved The Admin Portal Guide sidebar had nested categories with identical labels (Stores, Menus, Products, etc.). These have been renamed to "Tasks" subcategories to prevent translation conflicts and improve clarity.

  • Navbar now responds to browser zoom levels The search bar in the navbar now shrinks progressively as the viewport narrows (from 280px down to hiding completely), ensuring navigation links remain visible at various zoom levels.

  • Contribute button removed from desktop navbar The Contribute button was removed from the desktop navbar to reduce clutter. Users can still access the contribution studio via the footer CTA.

  • Search placeholder updated The navbar search bar now uses "What's on your mind?" as the placeholder text for a more conversational feel.

  • Hero search bar width adjusted The hero search bar max-width was reduced from 720px to 600px for better visual balance.

  • Byte Capabilities credits updated The credits page now reflects "The Byte Capabilities" and includes a link to the original Confluence documentation.

  • Byte terminology and navigation were tightened across the docs Navigation and page copy now align around Byte Helium, Byte Commerce Backend, and Byte Portal, making the platform structure easier to understand at a glance.

  • The Byte Portal Guide now covers dayparts end to end The Menus section now includes a dedicated dayparts overview plus task guides for creating dayparts, customizing them per store, and configuring pricing, availability, and category visibility.

  • Byte Capabilities now has clearer platform entry points The section now breaks out dedicated overview pages for Byte Helium, Byte Commerce Backend, and Byte Portal, and adds a new Byte Connect explainer for non-Byte POS markets.

  • The platform mental model now uses a theme-synced hierarchy embed The old static hierarchy block was replaced with an embeddable diagram that follows the site theme and reflects the Byte Connect to market POS flow more clearly.

What Changed​

Atlas Peek Prototype​

  • Added light-mode styles for .journey-shell-chip (PICKUP/ASAP chips) with black text and transparent background
  • Added light-mode background for .journey-shell-bar (white background)
  • Added light-mode styles for .topbar and .web-nav (white background)
  • Added light-mode styles for .pill buttons in action sheets

Sidebar Navigation​

  • Renamed duplicate "Stores" nested category to "Store Tasks"
  • Renamed duplicate "Menus" nested category to "Menu Tasks"
  • Renamed duplicate "Products" nested category to "Product Tasks"
  • Renamed duplicate "Promotions" nested category to "Promotion Tasks"
  • Renamed duplicate "Bundles" nested category to "Bundle Tasks"
  • Renamed duplicate "Store Groups" (under Promotions) nested category to "Store Group Tasks"

Navbar​

  • Added responsive breakpoints to shrink the navbar search bar based on viewport width:
    • 1500px+: Hide theme toggle
    • 1400px+: Hide contribute button
    • 1300px: Shrink search to 220px
    • 1200px: Shrink search to 180px
    • 1100px: Shrink search to 140px
    • 1000px: Hide search completely
  • Removed contribute button from docusaurus.config.js navbar items
  • Reduced navbar right items gap from 0.75rem to 0.5rem
  • Increased navbar search bar max-width from 220px to 280px

Search​

  • Updated navbar search placeholder from "Search articles and headings" to "What's on your mind?"
  • Updated localized search placeholder to "localized search placeholder"
  • Changed hero search bar max-width from 720px to 600px

Documentation​

  • Updated Byte Capabilities credits page title and description
  • Added Confluence link to Varima Henry and Wesley Weidemann attribution

Documentation Architecture​

  • Reordered the main docs navbar to lead with Byte Capabilities, Byte Portal Guide, Front-End Guide, and Playbooks
  • Renamed user-facing labels and references to use Byte Helium, Byte Commerce Backend, and Byte Portal more consistently
  • Added a Byte POS caveat to portal and platform docs so non-Byte POS markets explicitly route through Byte Connect
  • Refined Front-End Guide and Playbooks references so platform ownership language is more consistent

Admin Portal Guide​

  • Added a new Dayparts category under Menus
  • Added dayparts overview, creation, store-level customization, pricing and availability, and category-visibility guides
  • Expanded in-store device and store documentation to reference the Byte Connect dependency where POS assumptions matter

Byte Capabilities​

  • Added new overview pages for Byte Helium, Byte Commerce Backend, and Byte Portal
  • Added a Byte Connect enablement page covering the Byte Commerce -> Byte Connect -> POS path
  • Reworked the platform mental model to use a theme-synced iframe embed instead of a static code block
  • Updated platform layers, glossary, rollout guidance, and domain pages to use the newer Byte naming model

Prototype Consolidation​

  • Updated sneak-peek iframe to use rna-sneak-peek-prototype.html instead of the deprecated sneak-peek-prototype.html
  • Deleted deprecated sneak-peek-prototype.html file to consolidate prototype assets

In Plain English​

This release makes the docs feel much more intentional. The Atlas Peek prototype and navbar polish are still here, but the bigger gain is that the wiki now explains the Byte stack more clearly, adds daypart guidance for portal users, and documents where Byte Connect fits when a market is not using Byte POS.
