Release 2026.03.20

This release makes the architecture experience easier to explore, completes a major localized localisation pass, and improves requirement traceability across Byte capability documentation.

ExperiencePinned architecture notes stay visible while people keep browsing the diagram.
Coveragelocalized content now spans the full docs set, including explorer UI and release notes.
TraceabilityFRD references in Byte capability docs now deep-link back to source requirements when mapped.
Key highlights

The story first

Pinned notes now behave like a real companion panel

Users can pin a note, keep it visible while the architecture map scrolls underneath, and swap to another box without losing pinned mode.

Diagram UXHigher scanability

localized is now much more complete across Atlas

Every English documentation page now has a localized counterpart, and the platform explorer itself now opens with localized shell and architecture copy.

LocalizationMarket readiness

Technical source references are easier to trust

Byte capability technical source blocks now link individual Helium, Portal, and Commerce Backend references back to source requirements where exact mappings were available.

TraceabilityFRD linking

Dayparts now reads like a first-class Admin Portal domain

Daypart documentation moved out from under Menus and now sits alongside other top-level Admin Portal guide areas.

Information architectureAdmin Portal Guide
Release themes

Grouped instead of flat

Architecture explorer usability

The system architecture view now supports a more stable “pin and compare” workflow, and the embedded sizing chain expands correctly so pinned content is not clipped inside the docs shell.

localized localisation coverage

The release closes the remaining localized coverage gaps across docs, navigation, footer, homepage, release notes, and the embedded platform explorer.

Source-linked documentation quality

Capability pages are now more useful as a research surface because mapped FRD references can be opened from the technical source blocks directly.

Full change log

Open the lower-level detail when you need it

System architecture diagram
  • Restored the Pin this note interaction so hover and click still preview a box before pinning.
  • Updated pinned notes to behave like a persistent on-screen overlay while the underlying diagram keeps scrolling.
  • Kept pinned mode active when switching between boxes so users can compare systems without re-pinning.
Embedded explorer sizing
  • Added height reporting from the architecture diagram to the platform explorer wrapper.
  • Added height reporting from the platform explorer back to the docs iframe component.
  • Updated the mental model page embed to listen for live height changes from the explorer.
localized localisation

  • Added localized versions of all previously missing docs, including Byte Capabilities domains, enablement guides, references, release notes, and daypart documentation.

  • Updated localized navbar, footer, homepage, What's New, Contribute, and Atlas Peek copy to reduce English fallback text.
  • Regenerated localized search index content so the new translated docs are discoverable.
Platform explorer localisation
  • Updated the localized mental model page to pass locale into the platform explorer embed.
  • Localized the platform explorer shell so the localized page shows localized headings, tabs, panel labels, and footer copy.
  • Localized the embedded system architecture view with localized labels, legend text, panel UI, and translated box descriptions.
Byte capability source linking
  • Linked explicit FRD-HEL references in Byte capability technical source sections directly to their matching Helium requirements anchors.
  • Expanded compact Helium FRD ranges into individual linked references where needed so each requirement can be opened directly.
  • Linked approved FRD-BCOM and normalized FRD-PORTAL references where exact mappings were available, while leaving unmapped codes as plain text.
Admin Portal guide
  • Promoted Dayparts to a top-level Admin Portal Guide section instead of keeping it nested under Menus.
  • Re-homed daypart overview, definition, store-level customization, pricing and availability, and category-visibility guides under the new Dayparts section.
  • Updated the Admin Portal Guide landing page, sidebar, and related Stores and Menus entry points to point to the new canonical Dayparts hierarchy.
In plain English

The architecture diagram is much easier to use, the localized experience is much more complete, and the Byte capability docs are more credible as a source map. This release feels like both a usability pass and a coverage pass.
