Release 2026.03.22

This release cleans up the Brand Experience org chart, improves localized copy alignment across key docs, and makes the localized admin sidebar easier to scan with bilingual section labels.

ClarityCurrent-team reporting lines are easier to follow across Brand Experience, Engineering, and Support.
InteractionFocused team views now expand under Jim and let the broader overview collapse out of the way.
ThemeThe org chart now stays in sync with the site light and dark mode state.
Localizationlocalized homepage, docs, and admin navigation now track the English source more closely.
Key highlights

What changed most

Brand Experience reporting now reads like a real org view

Haley remains the clear center of the brand branch, with Mark, Justin, Wes, and Don presented in a way that makes peer relationships and dotted-line context easier to scan.

Org chart clarityBrand Experience

Focused views now replace the overview instead of competing with it

Clicking a focus CTA now hides the current-team overview and brings the chosen branch directly under Jim, making the detailed structure feel like a proper drill-down instead of a second disconnected section.

Interaction designCurrent Team

Engineering relationships are more explicit

The engineering panel now reflects the intended reporting chain more directly, including Jim above Matthew for context and the Jonathan, Brendan, Phong, Steve, and Don relationships laid out more clearly.

Engineering contextReporting lines

The embed now behaves like part of the product shell

The org chart iframe listens for theme changes from the docs site, updates live, and uses higher-contrast connector styling in dark mode so hierarchy lines stay visible.

localized navigation and copy are more literal and more usable

The localized homepage now reflects the English module naming, several high-traffic localized docs were aligned more directly with the English source, and the admin sidebar now uses bilingual labels like Stores / 店舗 for faster scanning.

localized localizationSidebar clarity
Release themes

What this release was really about

Hierarchy over decoration

Several org-chart changes in this window were less about adding content and more about making the reporting structure legible at a glance, especially in the current-team view.

Focused exploration without visual overload

The current-team experience now behaves more like a revealable branch under Jim, which reduces page noise and keeps people oriented when they switch between Brand, Engineering, and Support.

Chart and shell now feel connected

The embedded org chart now picks up theme state from the parent docs site, so the visual experience feels intentional rather than like a detached static asset.

Localization now favors source fidelity over loose adaptation

The localized experience in this release moves closer to one-for-one terminology and structure, especially for homepage labels, troubleshooting guidance, and admin navigation.

Full change log

Lower-level detail

Brand Experience org chart
  • Updated titles and roles across the Brand Experience branch, including Haley and Mark.
  • Adjusted Don’s position and relationship treatment so the current brand view reads more consistently.
  • Removed leftover background effects and improved connector contrast in dark mode.
Current-team interaction model
  • Changed focus CTA behavior so selecting a branch hides the overview and swaps the focused structure directly under Jim.
  • Added a reset path so clicking Current Team returns to the whole structure.
  • Removed redundant overview connector guides that were adding noise without helping orientation.
Engineering and support views
  • Clarified current engineering reporting context for Matthew, Jonathan, Brendan, Phong, Steve, and Don.
  • Fixed focus-panel structure so the branch tabs render as intended.
  • Cleaned up future-state cards and support grouping to keep the chart aligned with the current narrative.
Brand Experience page cleanup
  • Removed duplicate explanatory copy so the page centers on the visual org chart.
  • Removed the extra written lists below the embed and kept the page lightweight.
  • Kept the embedded chart height synced back to the parent shell for cleaner layout.
localized localisation cleanup
  • Updated the localized homepage stats and summary so module naming now matches the English source more closely.
  • Changed localized admin sidebar labels to bilingual patterns such as Stores / 店舗, Menus / メニュー, and Dayparts / デイパート.
  • Cleaned up several high-traffic localized docs where older translation passes had left behind mismatched product names or partially untranslated instructional text.
In plain English

This was a cleanup-and-clarity release. The org chart is easier to trust, the localized navigation is easier to scan, and the site reads more consistently across English and localized where people compare the two side by side.
