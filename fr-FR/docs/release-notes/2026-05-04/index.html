Release 2026.05.04

This release promotes the current KFC Australia prototype into Atlas Wiki, replaces the older hosted sneak-peek build cleanly, and adds better review controls so stakeholders can restart the journey or switch into a prototype-only fullscreen mode without losing context.

PrototypeThe hosted Atlas Peek experience now matches the current local KFC AU prototype instead of an older partial build.
DataThe AU menu-driven prototype now runs with the promoted runtime data file and required hosted font assets.
Review UXReaders can now restart the journey and open a prototype-only fullscreen review mode without clutter inside the phone frame.
CleanupLegacy hosted prototype data was removed so the wiki no longer mixes old and new sneak-peek logic.
retired market sunsetThe retired retired market locale and its locale-specific assets were removed so Atlas Wiki now runs as a leaner single-locale site.
Key highlights

What changed most

The hosted prototype now reflects the real current AU state

The old hosted sneak-peek experience was replaced with the current local KFC Australia prototype HTML rather than patched forward incrementally, so the wiki now shows the same cart, checkout WIP treatment, and menu-driven ordering behavior that exists in the active local working version.

Clean promotionKFC AU prototype

Supporting runtime assets were promoted with it

The AU menu data file and the Kentucky Fried font assets required by the prototype are now hosted in the wiki repo, which removes the gap between local-only dependencies and the published GitHub Pages version.

Menu dataFont hosting

Prototype review is easier to control

The sneak-peek page now includes a restart action outside the phone shell and a prototype-only fullscreen review mode, so readers can recover from deep checkout states and focus on the app flow without the rest of the wiki getting in the way.

Restart journeyFullscreen review

The route now behaves consistently in dev and hosted builds

The sneak-peek wrapper now points at the real hosted prototype asset path and uses a message-based reset hook, which avoids the earlier mismatch where the iframe could render differently between local development and the built site.

Pathing fixEmbed stability

The retired locale was removed cleanly

The old locale tree, retired search index output, locale-specific build wiring, and retired market-only hosted assets were removed so the wiki now reflects the active programme footprint instead of carrying inactive locale overhead.

Locale retirementBuild simplification
Release themes

What this release was really about

Promotion over hybridization

The main goal in this window was not to preserve every older hosted prototype affordance. It was to promote the actual current AU prototype state into the wiki cleanly so the review surface is trustworthy again.

Review loops should not dead-end at checkout

Readers often arrive at later prototype states and then need a clean way back to the beginning. Restart and fullscreen controls now make that review loop much more usable for walkthroughs and stakeholder demos.

Hosted prototypes need their real dependencies

This release closes the gap between local experimentation and the published wiki by promoting the data and font files the AU prototype actually depends on at runtime.

Retired market content should leave the stack

retired market has exited the programme, so the wiki should no longer build, index, or ship a second locale that no longer serves an active market audience.

Full change log

Lower-level detail

Atlas Peek prototype promotion
  • Replaced the hosted sneak-peek HTML with the current local KFC AU prototype implementation.
  • Updated the sneak-peek route wrapper so the iframe uses the correct hosted prototype asset path.
  • Kept the route as a review surface while removing stale wrapper behavior that depended on the older hosted build.
Runtime dependencies
  • Added the promoted kfc-au-menu-data.js runtime file to the hosted wiki.
  • Added the Kentucky Fried Sans, Serif, and Stripes WOFF/WOFF2 assets required by the prototype.
  • Removed the older hosted-only prototype data file so the AU flow no longer mixes incompatible runtime sources.
Prototype review controls
  • Added an external Restart Journey control that resets saved prototype state through a message-based hook.
  • Added a prototype-only fullscreen mode for /sneak-peek so readers can focus on the app flow without surrounding wiki chrome.
  • Moved restart control outside the phone frame so the prototype UI stays visually clean during demos and reviews.
Search and build outputs
  • Refreshed the English search index as part of the release build.
  • Verified the updated prototype route through local build output before promotion to GitHub Pages.
retired market locale removal
  • Removed the retired locale locale tree and locale-specific hosted assets that were no longer needed.
  • Removed locale-specific app copy, search wiring, and regression checks tied to the retired retired market experience.
  • Cut roughly 83.2 MB of known locale-specific repo and static footprint from the wiki.
In plain English

This was a trust-the-demo and trim-the-dead-weight release. Atlas Peek now reflects the real current AU prototype, it is easier to review without getting stuck at checkout, and the wiki no longer carries a retired retired market locale.
